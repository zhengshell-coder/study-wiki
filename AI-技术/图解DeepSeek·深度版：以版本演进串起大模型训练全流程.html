一个核心问题：大模型到底在"学"什么？

在开始之前，先建立整个领域最根本的直觉。语言模型只做一件事：给定前文，预测下一个词。就这么一个看似无聊的任务，当模型足够大、数据足够多时，"语法、知识、推理、代码"全都会从中涌现。

P(下一个词 | 前文) → 学会这门任务，就学会了语言蕴含的一切规律
生活类比

想象一个学生读了整个互联网的所有文本。你给他半句"中国的首都是"，他不用查字典就知道下一个字大概率是"北"，再下一个是"京"。这种"接龙"能力练到极致，他不仅记住了事实，还学会了推理——因为推理本身也体现在文本的"下一句该说什么"里。大模型就是这位学生的数学化版本。

本指南要回答的核心问题是：DeepSeek 是怎样把"预测下一个词"这件事，做到又强又便宜又开源的？答案分布在主流水线的每一环里。我们一环一环来看。

为什么要用"三维视角"理解大模型

大模型内部没有"文字"，只有高维空间中的向量——V3 的每个词在模型内部是一个 7168 维的坐标点。一切技术概念本质上都是对这个空间做的几何操作：训练是在空间里"搬点"，注意力是在点之间"连线加权"，MLA 压缩是"把点云拍扁到低维面上"，MoE 是"把空间划成责任区"。

人类无法想象 7168 维，但三维是我们能亲眼看见的最高维度，而这些几何直觉从 3 维推广到 7168 维时本质不变。所以本章先用两个可拖拽的三维场景建立"世界观"，再给出六大理念的全书索引——之后各章遇到任何复杂技术，你都可以回到这里找到它的几何原型。

3D 交互理念一 · 语义即几何：词向量空间
✦ 按住拖拽旋转视角。模型"读入"任何文字的第一步，就是把它变成空间中的一个点（Embedding）。训练之后奇妙的结构自动出现：意思相近的词彼此靠近（动物一团、代码一团、情感一团）；更惊人的是"关系"变成了"方向"——从"男"指向"女"的箭头，和从"王"指向"后"的箭头几乎平行，即经典的 王 − 男 + 女 ≈ 后。理解了"语义=几何"，后面所有技术就都有了着力点：模型的一切计算，都是在这个空间里移动、比较、汇聚这些点。
3D 交互理念二 · 注意力即聚光：按相关性分配光束
✦ 按住拖拽旋转视角。句子"猫追老鼠，因为它饿了"里，"它"指谁？模型的解法：让"它"升起来当 Query（探照灯），向句中每个词投射光束，光束粗细 = 注意力权重（由 Query 与各词 Key 的相似度经 softmax 算出）。可以看到"它"的光束强烈地打在"猫"上——指代就这样被解决了。切换不同 Query 会看到每个词关注的对象各不相同；真实模型里，每个词都要当一次 Query，每层还有 128 个头同时打光（各找各的关系）。这盏"聚光灯"就是第二章注意力机制、以及 MLA/DSA 一切优化的原型。

③ 训练 = 地形下山

损失函数是地形，参数是位置，梯度是坡度，学习率是步幅——整个训练就是在 6710 亿维地形里找最低谷。

④ 压缩 = 降维投影

高维数据往往贴着低维"薄面"分布（冗余），把点云拍扁到面上，只存面内坐标。MLA/DSA/CSA 全部生长于此。

⑤ 稀疏 = 分诊路由

不是所有参数都该为每个输入工作。把空间划成责任区，按需惊动少数专家——容量与算力从此解耦。

⑥ 并行 = 高维切块

训练任务是一个可以沿"数据/层/专家"多个维度切分的立方体，切法的选择 = 分布式策略的全部艺术。

DeepSeek 的位置：这六条理念是全行业共享的"物理定律"，DeepSeek 的贡献在于把其中的压缩（MLA）、稀疏（DeepSeekMoE）、下山的工程化（FP8/DualPipe）、以及"下山信号可以极简"（GRPO 纯 RL）做到了行业极致，并全部开源。技术细节会变，这些几何直觉不会——它们是你读完本书后真正该带走的东西。
第一章 · 神经网络、训练循环与 Transformer

第一层·直觉：什么是"参数"和"训练"？

调音台的类比

把神经网络想象成一个有几十亿个旋钮的巨型调音台。输入一段文字，信号流过这些旋钮，最后输出一个预测。一开始所有旋钮都是乱拧的，预测也是乱的。"训练"就是反复微调这些旋钮，让预测越来越准。每一个"旋钮"就是一个参数（parameter），所谓"67B 模型"就是有 670 亿个这样的旋钮。

把旋钮组织起来的最小单元叫神经元：它接收若干输入，各自乘一个权重（旋钮值），加起来，再过一个非线性"激活函数"输出。很多神经元排成一层，很多层摞起来，就是深度神经网络。"深度学习"的"深"就是层数多。

第二层·机制：训练循环（最该理解的一张图）

不管模型多大、多复杂，训练时它都在反复执行这四步循环。这是理解一切训练技术的地基：

直觉
做卷子→对答案→想哪错了→下次别犯
机制
前向→Loss→反向求导→AdamW更新
工程
混合精度+梯度累积+分布式同步
关键概念：学习率

优化器每步"调旋钮"的幅度叫学习率。太大→震荡发散；太小→学得太慢。DeepSeek-V3 用预热+余弦衰减策略：开头小步试探，中间大步前进，结尾慢慢收尾。这个调度对训练稳定性极其关键（第五章详解）。

3D 交互损失地形与梯度下降 · 三档学习率对比
✦ 按住拖拽旋转视角。这张"地形图"是损失函数的可视化：水平面上每个点代表一组参数取值（真实模型是 6710 亿维，这里压成 2 维），高度代表这组参数下模型"错得有多离谱"。训练 = 小球从随机起点出发，沿最陡下坡方向（负梯度）一步步滚向谷底。三档学习率各演示一种命运：太小——每步只挪一点点，磨到收敛要 900+ 步，算力成本翻约 9 倍；合适——约 100 步稳稳落进全局最优；太大——平缓地带看似正常，一进陡谷每步都跨过头，在两壁间弹跳（红色折线），永远到不了底。这正是第四章"训练稳定性"和"学习率调度"要解决的问题。

为什么是 Transformer？从 RNN 的痛点说起

2017 年之前，处理序列的主力是 RNN/LSTM。它有一个致命问题：必须一个词一个词地算，第 1000 个词的计算依赖第 999 个词的结果，无法并行。而且梯度传着传着就消失了，远处的信息记不住。

Google 在 2017 年提出 Transformer（《Attention Is All You Need》），核心是自注意力机制：每个词可以直接看到序列里所有其他词，一步到位算出关系，天然并行。这成为之后所有大模型（GPT、LLaMA、DeepSeek）的统一底座。

时代代表核心思想痛点
统计N-gram数前 N 个词的共现频率N>5 数据稀疏，记不住远处
循环网络RNN/LSTM逐步传递隐状态不能并行，梯度消失
当前标准Transformer自注意力，全局直连O(n²) 复杂度（可优化）

Scaling Laws：为什么"大"本身就是技术

2020 年 OpenAI 发现：模型损失与参数量 N、数据量 D、算力 C 之间存在可预测的幂律。意思是——只要按比例放大三者，性能就能可预期地提升。这条规律把 AI 从"靠灵感"变成了"靠工程"。

L(N) ≈ (N_c / N)α　　损失随参数量幂律下降

DeepSeek 的策略不是"砸最多算力"，而是在 Scaling Laws 框架内，把每一分算力的效率榨到极致——这贯穿了它后续每一代的设计哲学。

第一层·直觉：注意力到底在干什么

图书馆找书的类比

你带着一个查询（Query）——"我想找讲注意力的书"。每本书有一个键（Key）——书名/标签。你拿 Query 去和每本书的 Key 比对，相关性高的书（Value）对你最终的决定影响更大。注意力机制就是这个过程的数学化：按相关性加权汇总信息。

在"自"注意力里，Query、Key、Value 都来自同一句话的不同变换。每个词都用这种方式去"看"其他所有词，决定从谁那里吸收多少信息。

第二层·机制：Self-Attention 六步计算

Attention(Q, K, V) = softmax(QKᵀ / √dk) · V

多头注意力：一个团队，多种视角

一个注意力头只能学一种"关注模式"。多头让模型同时从多个子空间学习——有的头看语法、有的看语义、有的看位置。V3 有 128 层、每层 128 头，像 128 个分析师各看一个角度再汇总。

MultiHead = Concat(head₁, ..., headₕ) · WO

RoPE 旋转位置编码：用"旋转"表示位置

注意力本身不知道词的顺序（"猫咬狗"和"狗咬猫"对它是同一组词）。必须注入位置信息。DeepSeek 用 RoPE（旋转位置编码）——它的思想很优雅：把 Q 和 K 向量两两配对，按词的位置旋转不同角度。

RoPE 的妙处

两个词做点积时，旋转角度会相减——所以 Q 和 K 的点积只依赖相对位置而非绝对位置。这正符合语言直觉："我爱你"里"爱"和"你"的关系，跟它们在第几句无关，只跟它们隔几个词有关。RoPE 不增加参数，且为后续扩展上下文（YaRN 调频率）留了口子。

KV Cache：推理的"内存炸弹"

模型生成文本时是自回归的——每生成一个词就把它接回去继续生成。如果每次重算所有词的 K/V 太浪费。所以把算过的 K/V 缓存起来，新词只算自己的。

问题：标准 MHA 的 KV Cache 巨大

每个 token 要缓存所有头的 K 和 V：
KV Cache = 2 × n_heads × d_head × n_layers × seq_len × 精度
一个 70B 级模型跑 128K 上下文，纯 MHA 单请求 KV Cache 可达 300GB+；即便用了 GQA（8 组共享）也要 约 40GB——比模型权重还占地方。这就是必须进一步压缩的根本原因。

第三层·工程：MLA 低秩压缩到底怎么做到的

这是 DeepSeek 从 V2 起的核心竞争力。直觉：标准 MHA 要缓存所有头的完整 K/V，但这些信息里有大量冗余——可以用一个低维"潜在向量"高效表示。

图像压缩的类比（理解"低秩"）

一张 1024×1024 的图有 100 万个像素，但往往可以用几十个"基础模式"的加权组合来近似重建（这就是 JPEG、SVD 降维的原理）。MLA 做的也是这件事：不存完整的 K/V，而是存一个几百维的压缩向量 c，需要时再用"上投影"把 K/V 恢复出来。压缩有损，但损失小到几乎不影响效果。

压缩：c = Wdown · x　（7168维 → 512维）
恢复：K = WupK·c，　V = WupV·c　（512维 → 各头维度）

MLA 使 V2 的推理 KV Cache 减少 93.3%，训练成本降 42.5%，吞吐提升 5.76 倍。这是 DeepSeek API 能做到行业最低价的技术基石。

RoPE 解耦：一个必须处理的冲突

RoPE 要在注意力前作用于 Q/K，但压缩后的 c 无法直接旋转。DeepSeek 的方案是解耦式 RoPE：主走 MLA 压缩路径（位置无关），额外留一小段"RoPE 专用维度"单独旋转。看似细节，实现门槛很高——V3.2 开源后还专门修过 RoPE 布局（Indexer 非交错 / MLA 交错）的 bug。

MLA 的深层意义：它证明注意力 KV 信息存在大量冗余，可用低维潜在空间高效表示。这个洞察成为 DeepSeek 后续一系列注意力创新（DSA/CSA/HCA）的起点——都是在"信息可以压缩"这个命题上做文章。
3D 交互MLA 低秩压缩：为什么"压得掉"
✦ 按住拖拽旋转视角。每个亮点是一个 token 的 K/V 向量（真实是 32768 维，这里画成 3 维）。旋转后你会发现：这团点云并不是随机撒满空间的，而是紧贴着一张倾斜的"薄面"分布——这就是"低秩"的含义。既然点都住在面附近，就不必存三维坐标，只存"在面上的位置"（= 512 维潜在向量 c）即可；用时再沿面恢复，误差（黄色短线）小到不影响效果。压缩 93.3% 的底气，来自数据本身的冗余结构。

第一层·直觉：为什么需要 MoE

医院的类比

稠密模型像一个"全科医生"，什么病都自己看，但要看遍所有知识才能下结论，效率低。MoE 像一家大医院：有几百个专科医生（专家），来了病人先由分诊台（Router）判断该挂哪个科，只请相关几位医生会诊。医院整体很大（知识全），但每次看病只动用少数医生（算得快）。

稠密模型的根本矛盾：知识容量和计算成本都随参数线性增长。MoE 把两者解耦——总参数很大（容量大），但每个 token 只激活一小部分（成本低）。

y = Σᵢ G(x)ᵢ · Eᵢ(x)　，G(x)=softmax(Top-K(x·Wg))

Router 算每个专家与输入的匹配分，选 Top-K 个，加权求和。V3 每个 token 激活 8 个路由专家 + 1 个共享专家（V2 时代为 6 路由 + 2 共享）。

第二层·机制：DeepSeekMoE 的两个关键差异

  • 细粒度专家：256 个小专家代替 8 个大专家。组合数指数级增加，每个 token 能更精确"拼装"所需能力——像用乐高小块而非整块大积木。
  • 共享专家：始终激活、处理通用知识（V2 用 2 个，V3 起精简为 1 个），保证路由失败时基础能力不掉。
3D 交互MoE 路由：token 只惊动少数专家
✦ 按住拖拽旋转视角。白色小球是逐个到来的 token。每个 token 先经过顶部的 Router（分诊台）打分，然后只飞向被选中的 2 个路由专家（演示把 256 个简化为 16 个，真实 V3 是 256 选 8）；绿色的共享专家每个 token 都会经过。注意观察：不同 token 点亮的专家组合不同（数学 token 和代码 token 走不同的科室），而任一时刻大部分专家是暗的——这就是"1.6T 总参数、只算 49B"的秘密：容量与算力解耦。

第三层·工程：无辅助损失负载均衡

MoE 训练的经典毛病：负载不均衡。大多数 token 涌向少数热门专家，冷门专家得不到训练，模型退化。

传统做法是加一个辅助损失惩罚不均衡——但它会干扰主训练目标，模型变笨。DeepSeek-V3 的创新：不用损失，用动态偏置。

选择概率 = softmax(x·Wg + b)　，b 按负载动态加减
餐厅调度的类比

不是给忙的厨师罚款（辅助损失，伤积极性），而是给闲的厨师挂"今日推荐"牌（提高偏置 b），把客人引流过去。忙的厨师摘掉牌子（降低 b）。"软调节"既均衡又不干扰主业。

效果：256 个专家负载均衡良好，模型质量反而优于用辅助损失的版本。这是 V3 能把 MoE 训稳的关键工程贡献之一。

MoE 的隐藏难点：All-to-All 通信

256 个专家分布在几百张 GPU 上。每个 token 算出要找哪几个专家后，必须把数据发给存有这些专家的卡，算完再收回来。这种"所有卡两两交换数据"的通信叫 All-to-All，是 MoE 训练的最大瓶颈。

这就是 DeepEP 存在的理由：DeepSeek 开源的 DeepEP 通信库，专门优化 MoE 的 All-to-All——低延迟 Kernel（推理）+ 高吞吐 Kernel（训练），把通信和计算重叠起来隐藏延迟。没有这类优化，MoE 的算力利用率会跌到 30% 以下。

第四章 · 预训练全流程：数据·目标·训练循环
数据管线 · 分词 · NTP→MTP · 训练超参 · 稳定性 · 上下文扩展
V1: 2T token · V3: 14.8T token + MTP + 全程无尖峰

数据：大模型的"食物"，质量>数量

预训练的第一件事是喂数据。数据决定能力上限。V3 在 14.8T token 上训练，来源大致：

  • 网页文本：CommonCrawl 抓取，质量过滤
  • 图书/论文：高质量长文本，世界知识
  • 代码：GitHub 等，占比约 30%，提升编程与推理
  • 数学数据：数学推理与科学文献
数据质量的"看不见的功夫"

DeepSeek 在去重、去毒、质量打分上投入巨大。好数据策略能用更少 token 训出更好模型——这比"堆数据"更考验工程。V1→V3 token 数从 2T 涨到 14.8T，但更重要的是数据质量大幅提升。

分词：让模型"看懂"文字

计算机只认数字。第一步是把文本切成 token（标记）再映射成 ID。DeepSeek 用 BPE（Byte-Pair Encoding）：统计所有字符对频率，反复合并最高频的对，形成词表。

# BPE 反复合并最高频字符对 "深度学习改变世界" -> [3276, 18922, 5031, 8872] "def fibonacci(n):" -> [8912, 44321, 11, 89] # 词表 ~128K，覆盖中英混排与代码

词表大小是权衡：太小→序列变长算得慢；太大→嵌入矩阵占显存。128K 对中英代码混合是较优解。

训练目标：从"下一个词"到"未来多个词"

标准目标是 NTP（Next Token Prediction）：给定前文预测下一个 token，用交叉熵损失。

L = -Σ log P(真实下一个 token | 前文)

V3 的创新：MTP（Multi-Token Prediction）——在主干上叠加多个辅助预测头，每个头负责预测往后第 k 个位置。一次前向，同时学多个未来 token。

MTP 两个好处：(1) 训练学到更丰富序列信息，利好代码与数学；(2) 推理时辅助头可当"草稿模型"做推测解码，主模型快速验证，推理加速 2-3 倍。

训练循环的工程细节：把循环跑稳跑快

第一章讲的四步循环，在 V3 这种规模下要跑约 2.8 个月、2048 张 H800。要让这个循环既快又稳，关键超参与技巧：

  • 优化器 AdamW：带权重衰减的自适应梯度方法，是大模型标配
  • Batch Size 逐步放大：开头小 batch 试探，稳定后放大提升吞吐
  • 梯度裁剪：梯度范数超阈值就缩放，防梯度爆炸
  • 混合精度：BF16 主干 + FP8 矩阵乘（第五章详解）
  • Checkpoint 恢复：定期存档，硬件故障可回滚
一个惊人的成就：V3 整个训练过程没有出现任何不可恢复的损失尖峰或回滚。对 671B/2048 GPU/2.8 月的训练，这种稳定性业内极罕见。靠的是精心设计的容错、数值稳定性、以及 FP8 的精细精度管理。

第五章 · 分布式训练：并行·DualPipe·FP8
显存预算 · 四种并行 · DualPipe 双向流水 · FP8 混合精度 · 基础设施开源
V3 首创：671B 规模 FP8 训练 + 无 TP 并行 + DualPipe

为什么必须分布式：显存预算

671B 参数，一张 H100（80GB）连权重都放不下。训练时显存要装四样东西：

四种并行策略

V3 不用张量并行（TP）就训成了 671B——这在同规模里非常罕见。TP 每步都要 AllReduce 通信，是瓶颈。DeepSeek 靠精心优化内存布局和通信模式绕开了它，是重大工程成就。

3D 交互并行 = 沿不同维度"切"训练任务
✦ 按住拖拽旋转视角。把整个训练任务想象成一个立方体，三条棱分别是三种"可切的维度"：数据批次（DP）、模型层（PP）、专家（EP）。每种并行策略就是沿一条棱把立方体切成片，每片交给一组 GPU；点击"组合并行"看三刀同时切下去的效果——每个小块 = 一张 GPU 的职责范围，DP×PP×EP 相乘就是总卡数（V3：跨这三维组合出 2048 张 H800，刻意不切"张量"这第四刀以省通信）。旋转立方体，体会"并行策略的本质是给高维任务选切法"。

DualPipe：几乎消除流水线气泡

流水线并行有固有的"气泡"：某些 GPU 等其他卡结果时闲置，传统约浪费 20-30%。DualPipe 用双向流水：数据从两端同时输入相向而行，前向计算和反向通信重叠，几乎消除气泡。

FP8 混合精度：把算力榨到极致

降低数值精度是省显存+加速的核心。V3 是首个在 671B 超大规模上验证 FP8 训练的团队。

精度管理策略（不是全部用 FP8，而是混合）：矩阵乘法 GEMM 用 FP8（计算量最大）；累加用 FP32（保精度）；LayerNorm/Softmax 保留 BF16/FP32；梯度通信用 BF16。还要做分块缩放（tile-wise scaling）——每个小块动态选合适缩放因子，弥补 FP8 表示范围窄的问题。

成本对比：V3 的 14.8T token 训练仅花 2.788M H800 GPU 小时，按市场价约 557 万美元。对比 GPT-4 训练成本估计超 6300 万美元——DeepSeek 用不到 1/10 的成本训出接近 GPT-4 水平的模型。FP8 是其中最关键的节流手段。

第六章 · 长上下文：从 128K 到 1M

核心挑战：注意力的 O(n²) 诅咒

标准注意力的计算和显存都随序列长度平方增长。从 128K 到 1M，计算量涨约 64 倍。

上下文注意力计算量体验
8K1×流畅
32K16×可接受
128K256×需优化
1M16,384×必须专项优化
关键洞察：并非所有 token 间的注意力都重要。自然语言大量冗余，多数 token 对的注意力权重很低，可安全跳过。这是所有稀疏注意力的理论基础。

FlashAttention：先把显存降到 O(n)

FlashAttention 不改计算量，但通过分块（tiling）把显存从 O(n²) 降到 O(n)：把 Q/K/V 切成小块，在 GPU 高速 SRAM 里完成小块注意力，避免把巨大的 n×n 注意力矩阵写回慢速 HBM 显存。它是 IO 优化，不是算力优化。

DSA：DeepSeek 稀疏注意力（V3.2）

FlashAttention 只省显存不省算力。V3.2 引入 DSA：用轻量级 Indexer 模块预测哪些 KV 位置重要，只对 Top-K 重要位置做完整注意力。

DSA 的硬件亲和性

Indexer 的预测与 Hopper GPU 的稀疏计算支持配合，实现计算-通信协同。DeepSeek 开源了 CUDA 和 TileLang 两版内核。DSA 是"信息可压缩"洞察在注意力上的直接延伸——和 MLA 一脉相承。

V4 的 CSA + HCA：让 1M 上下文成真

CSA（压缩稀疏注意力）

  • 压缩率 m=4：每 4 token 合并成 1 个 KV
  • 对压缩后 KV 做 DSA 稀疏注意力
  • 保留 128 窗口 token 不压缩（保局部精度）
  • 双重压缩流（交叠窗口）保边界
  • 定位：精细的长程 token 级注意力

HCA（高压缩注意力）

  • 压缩率 m'=128：极高压缩比
  • 不做稀疏化，保持密集
  • 提供粗粒度全局上下文视图
  • 一个序列流（高压缩+滑动窗口）
  • 定位：极低成本的全局长程记忆
混合哲学：CSA 负责"精细"（哪几个 token 重要），HCA 负责"粗略"（整体在讨论什么主题）。配合 MLA 低秩压缩，三重机制把 V4 的 1M 推理 FLOPs 降到 V3.2 的 27%，KV Cache 降到 10%。

第七章 · 后训练：SFT·GRPO·R1·蒸馏
从续写器到助手 · SFT · RLHF→GRPO · R1 推理涌现 · 四阶段 · 蒸馏
R1: 纯 RL 涌现推理（发表于 Nature）· V3.1: 双模式 · V4: 专家优先

为什么需要后训练？

预训练后的模型是个强大的"文本续写器"——擅长预测下一个词，但不擅长"回答问题"。后训练把它变成有用的 AI 助手。

SFT：教会模型"听指令"

用高质量"指令-回复"对微调模型。数量不需多（数千到数万条），但质量要求极高。SFT 后模型从"乱续写"变成"能回答问题"，但风格可能啰嗦或不安全——这要靠 RL 解决。

GRPO：DeepSeek 的强化学习创新

传统 PPO 要训一个 Critic（价值模型）估计"当前策略有多好"，又贵又难。DeepSeek 的 GRPO（Group Relative Policy Optimization）用组内相对比较消除了 Critic。

基线 b = (1/G)·Σ r(yi)　组内平均奖励
优势 Ai = r(yi) - b　高于平均→正激励，低于→负激励
+ KL 惩罚项 β·KL(π‖πref) 防止偏离太远

R1：推理能力的"涌现"

DeepSeek-R1 证明了一个颠覆性结论：推理能力不一定需要人类标注的推理示范。通过纯 RL 和可验证奖励（答案对错），模型能自行发现反思、验证、分解等推理策略。该工作 2025 年 9 月发表于 Nature（vol 645，封面论文）——纯 RL 过程中，R1-Zero 在 AIME 2024 的 pass@1 从 15.6% 一路涨到 77.9%（自洽投票下 86.7%），只靠"最终答案对不对"这一个信号。

"Aha 时刻"：R1-Zero（纯 RL、无 SFT 冷启动）在训练中自发出现了一段行为——模型输出"Wait, let me reconsider..."（等等，让我重新想想）。这种自我反思不是人教的，是 RL 探索出来的。这是推理"涌现"的标志性瞬间。

DSpark 推测解码

V4 配套的推测解码方法（置信度调度 + 半自回归草稿），单用户生成速度较 MTP-1 基线再快 60–85%；训练/评估代码以 DeepSpec 框架开源（2026.06）。

推测解码原理：小模型（草稿头）一次快速猜出几个候选 token，大模型一次性并行验证这些候选。猜对的直接接受，猜错的从错误处重新生成。因为大模型验证 N 个 token 和生成 1 个 token 时间相近，所以能整体加速。V3 用 MTP 头做草稿是这个思路的 1.0；V4 的 DSpark 在此之上加了"半自回归草稿 + 按置信度动态决定猜多远"，猜得更长也更准。
部署形态：V4 官方推荐 vLLM 或 SGLang。vLLM 支持 MLA 的 PagedAttention、TP/PP、Chunked Prefill。部署 671B 通常需 8 张 H100(80GB) 起步。

Thinking vs Non-Thinking 双模式

从 V3.1 起 DeepSeek 支持两种推理模式，按需选择：

Thinking（思考）

  • 先内部推理再输出答案
  • 适合：数学、代码、复杂推理、Agent
  • 支持 reasoning_effort（high/max）
  • 质量高，延迟较长

Non-Thinking（直接答）

  • 直接输出，省略推理
  • 适合：日常对话、翻译、摘要
  • 响应快，token 消耗少
  • 成本更低

背后是重要的产品洞察：不是所有问题都需要深度推理。问"今天天气"不需要思维链。双模式让用户按需平衡性能与成本。

DeepSeek 的"基因"：为什么是它做成了

典型 AI 公司

  • 追求商业化营收
  • 闭源 + API 收费
  • 有限开源权重
  • 依赖外部算力
  • 研究服务产品

DeepSeek

  • 追求 AGI 研究突破
  • MIT 协议 + API 超低价
  • 全栈开源（权重+设施+内核）
  • 自研训练框架与通信库
  • 产品反哺研究
关键背景：DeepSeek 背靠量化对冲基金 High-Flyer，创始人梁文锋是数学与工程出身。量化投资的基因就是"用更少资源获更大回报"——这直接决定了 DeepSeek 对算法创新和工程效率的极致追求：不堆算力，而是优化每一分算力。这是 MLA/FP8/DualPipe 这些"省到极致"技术能从 DeepSeek 而非算力巨头手里诞生的深层原因。

四条行业启示

算法-工程协同设计

低成本不靠堆硬件，靠 MLA/DSA/FP8/DualPipe 等算法与工程的协同创新。每层优化都来自对问题的深刻理解。

开源可以超越闭源

V4-Pro 多维度达甚至超闭源模型。证明开源模式技术上完全可行，且因社区参与可能更快迭代。

推理可以涌现

R1 纯 RL 路线证明推理能力不一定需要人类示范。机器能从简单"对/错"信号中自行发现复杂推理策略。

Agent 是新前沿

V4 把 Agent 能力作为核心设计目标，验证了大模型从"对话工具"到"工作助手"的范式转变。

未来展望

推理深化

纯 RL 推理涌现继续深化，从"刻意触发"变为"自然融入"日常交互。

超长上下文普及

1M+ token 成标配，全量输入取代检索片段，改变 RAG/Agent/文档处理架构。

Agent 自主性

从对话到行动。V4 已展示 Agentic Coding，未来扩展到更多真实世界任务。

多模态融合

DeepSeek 已有 VL/OCR 多模态模型，V4 主模型纯文本，多模态能力开发中。

效率革命

保持能力同时持续降推理成本，让大模型真正无处不在。

在线学习

V4 技术报告提到为在线学习范式奠基——模型可能从交互中持续学习更新。
