大模型训练全过程：一张地图

在深入每个细节之前，先建立一张完整的地图：训练一个大模型，到底要经过哪些阶段？

今天我们用的 ChatGPT、Claude、文心、通义这类「会聊天、会写代码、会推理」的模型，本质上都是一个庞大的神经网络。它从「一堆随机数字」变成「能听懂人话的助手」，要经过一条很长的流水线。这套文档会沿着这条流水线，从最浅显的概念一路讲到工程细节。

先记住这条主线

整个训练可以拆成两大块：预训练（Pre-training）让模型「读完整个互联网，学会世界知识和语言规律」；后训练（Post-training）让模型「学会按人类期望的方式说话、做事、讲道理」。前者烧的是算力，后者拼的是数据质量和对齐方法。

完整流水线一览

① 数据采集、清洗、去重、配比
→
② 分词把文本切成 token
→
③ 预训练预测下一个 token
→
④ SFT指令微调
→
⑤ 对齐RLHF / DPO
→
⑥ 推理强化RLVR / 长思维链
→
⑦ 部署推理优化、上线

各阶段一句话职责

阶段在做什么产出主要成本
数据准备把几十 TB 的原始网页变成干净、去重、配比合理的训练语料高质量 token 池人力 + 存储 + CPU

SFT 监督微调用「指令→理想回答」示范，教模型听懂任务指令模型高质量标注数据

部署量化、KV Cache、并行，把模型跑得又快又便宜线上服务推理算力

过去 vs 现在：最重要的三个变化

之前（2018 年前后）
  • 每个任务单独训一个模型（情感分类一个、翻译一个）
  • 靠人工标注的小数据集
  • 模型几千万～几亿参数
  • 「预训练 + 针对任务微调」是主流范式
现在（2023 年之后）
  • 一个通用基座模型解决几乎所有任务
  • 预训练靠自监督，不需要人工标注
  • 模型百亿～万亿参数（含 MoE）
  • 「预训练 + 指令微调 + RL 对齐 + 推理强化」四段式

之前是怎么干的：从规则到神经网络

理解今天，必须先看清来路。NLP 走过了规则、统计、深度学习三个时代，每一次范式跃迁都源于一个旧瓶颈被打破。

「让机器理解语言」这件事，人类折腾了七十年。每个时代都有自己的核心方法，也有把它逼到尽头的瓶颈。看懂这条演化链，你就能理解为什么今天会是这个样子。

第一阶段：规则时代（1950s–1980s）

最早的思路很直觉：既然语言有语法，那就让语言学家把规则一条条写进程序。比如「主语 + 谓语 + 宾语」，再配一本词典。

核心瓶颈

语言的例外、歧义、口语千变万化，规则写不完，而且彼此冲突。一个稍微复杂的句子就能让系统崩溃。人工规则无法规模化。

第二阶段：统计时代（1990s–2010s）

转折点是：与其手写规则，不如从数据里统计概率。代表是 N-gram 语言模型——用前 N-1 个词预测下一个词的概率。机器翻译、语音识别都靠它吃饭。

P(下一个词 | 前文) ≈ P(下一个词 | 前 N-1 个词)
例：P("好" | "今天天气很") —— 只看有限的前几个词
核心瓶颈

① 维度灾难：N 一大，组合数爆炸，绝大多数词组在语料里从没出现过（数据稀疏）。② 记不住长距离依赖：N-gram 只能看前几个词，「这本书……我去年读过」这种跨度它抓不住。

第三阶段：神经网络与词向量（2013 起）

Word2Vec（2013）让每个词变成一个稠密向量，语义相近的词在空间里也相近（著名的「国王 - 男人 + 女人 ≈ 女王」）。随后 RNN / LSTM 登场，理论上能把任意长的历史压进一个「隐藏状态」里逐字处理。Seq2Seq + 注意力（2014–2015）撑起了第一代神经机器翻译。

核心瓶颈

RNN 必须一个词一个词顺序处理，无法并行，训练慢；序列一长还会梯度消失，「记忆」实际上传不远。这成了扩大规模的天花板。

第四阶段：Transformer 革命（2017）

论文《Attention Is All You Need》提出 Transformer，用自注意力（self-attention）彻底丢掉了循环结构。任意两个词可以直接「互相看到」，而且整个序列能并行计算。这一下同时解决了「长依赖」和「不能并行」两个老瓶颈，为「把模型和数据一起做大」铺平了道路。

第五阶段：预训练 + 大规模化（2018–2022）

BERT 路线（理解）

双向编码，擅长分类、抽取、检索类任务，「先预训练再针对任务微调」成为主流。

GPT 路线（生成）

单向自回归，专注「预测下一个词」。GPT-2、GPT-3 证明了一件事：只要把模型和数据同时放大，能力会突现。

GPT-3（2020，1750 亿参数）展示了「少样本/零样本」能力——不再为每个任务单独训练，给几个例子甚至一句话指令就能干活。范式开始从「为任务微调」转向「一个通用模型 + 提示词」。

第六阶段：对齐与对话（2022 至今）

纯预训练模型「博学但不听话」：它会续写、会胡说，却不一定回答你的问题。InstructGPT / ChatGPT（2022）引入 RLHF（基于人类反馈的强化学习），把模型对齐到「有用、诚实、无害」，这才有了今天的对话体验。2024–2025 又出现 推理模型（如 o1 风格）：用强化学习训练模型「先长篇思考再作答」，把数学、代码、逻辑能力推上新台阶。

一句话总结演化逻辑

每一代都是在解决上一代的瓶颈：规则写不完 → 用统计；统计记不住长程 → 用 RNN；RNN 不能并行 → 用 Transformer；模型博学但不听话 → 用 RLHF；会说话但不会深度推理 → 用推理强化。瓶颈驱动范式迁移，这就是这份文档后续章节的暗线。

数据：模型能力的真正上限

有句行话：「Garbage in, garbage out」。预训练 90% 的功夫花在数据上。模型见过什么样的数据，就有什么样的能力边界。

很多人以为训练大模型最难的是算法，其实在工业界，数据工程才是投入最大、最影响成败的环节。模型的知识、语气、偏见、能力上限，几乎全由训练数据决定。

数据从哪来

  • 网页爬取：Common Crawl 等大规模网页存档是主力来源，原始体量达 PB 级。
  • 高质量语料：维基百科、书籍、论文（arXiv、PubMed）、问答社区。
  • 代码：GitHub 等公开代码。代码能显著提升模型的推理和结构化能力。
  • 多语言 / 多模态：各语种文本，以及图文对（用于多模态模型）。

清洗流水线：从几十 TB 到几 TB

原始网页里充斥着导航栏、广告、乱码、垃圾内容。清洗是个层层过滤的漏斗：

抽正文去掉 HTML、模板、广告
→
语言过滤识别语种、剔除乱码
→
质量过滤规则 + 模型打分，删低质
→
去重文档级 / 句级去重
→
去污染剔除测试集、隐私、有害内容

为什么「去重」这么关键

互联网上同一篇内容可能被转载成千上万次。如果不去重，模型会过度记忆高频重复内容、浪费算力，泛化能力反而下降。研究反复证明：去重能在同等算力下显著提升效果。

过去 vs 现在：数据观念的转变

之前的做法
  • 「数据越多越好」，粗放堆量
  • 清洗主要靠简单规则（关键词黑名单、长度阈值）
  • 各来源简单混在一起
现在的做法
  • 「质量 > 数量」，宁缺毋滥
  • 用小模型给数据打质量分，甚至用大模型筛/改写数据
  • 精细设计数据配比与课程顺序（先易后难、训练后期提高高质量数据占比）
  • 大量使用合成数据（模型自己生成高质量样本）
合成数据：当前最热的方向之一

真实优质数据正在被「训练消耗殆尽」，于是业界越来越多用强模型生成教科书式的高质量数据来训练新模型（如「教科书级」语料能让小模型表现远超体量）。但要警惕模型崩溃（model collapse）——纯用模型生成数据反复自训，会让多样性退化。

进阶：为什么「质量 > 数量」在数学上成立

回想预训练的目标是「预测下一个词」。如果语料里塞满低质、矛盾、重复的文本，模型就在学习一个混乱的概率分布——它会同时被教「这种烂写法也对」。等于一边教好习惯一边教坏习惯，互相抵消，还白白消耗了昂贵的算力。

反过来，一条高质量样本提供的「干净信号」抵得过许多条噪声样本。著名实验显示，用「教科书级」的精选/合成数据训练，小模型能达到数倍体量模型的水平。这就是配比与课程（先广撒网打基础、后期提高优质与难样本占比）越来越被重视的原因——它在不加算力的前提下提升了每一份算力的产出。

核心瓶颈
  • 高质量数据枯竭：互联网上的优质人类文本是有限的，可能已接近用尽。
  • 数据污染：评测集泄漏进训练数据，会让跑分虚高、误导判断。
  • 版权与合规：训练数据的来源合法性日益受关注。
  • 配比是门玄学：代码、数学、多语言各放多少，缺乏理论，靠大量实验试错。

分词：模型其实不认识「字」

模型处理的不是文字，而是数字编号。文本如何切分成 token，直接影响效率、成本和多语言表现。

神经网络只会算数字。所以第一步要把文本切成一个个token（词元），再映射成整数 ID。一个 token 大致相当于英文里 3～4 个字符，或者一个常见词的一部分；中文里通常 1 个 token 约等于 1～2 个汉字。

为什么不直接按「字」或「词」切

按词切（早期）
  • 词表巨大（几十万），还总有没见过的「未登录词 OOV」
  • 遇到没见过的词只能用 [UNK] 占位，信息丢失
按字符切
  • 词表很小，没有 OOV 问题
  • 但序列变得超长，效率低，语义太碎

现在的主流：子词分词（Subword）

折中方案是子词：常见词作为整体（如 "the"），生僻词拆成片段（如 "tokenization" → "token" + "ization"）。这样既控制了词表大小（通常 3 万～15 万），又能拼出任何没见过的词。主流算法是 BPE（Byte-Pair Encoding） 及其字节级变体，从字节开始，不断把「最常一起出现的相邻片段」合并成新 token。

初始: l o w e r   (按字符/字节)
统计最高频相邻对 "e r" -> 合并
逐步合并: lo w er -> low er -> lower
最终词表里既有完整词，也有可拼装的片段

分词如何影响成本和公平

~4 字符
英文 1 token 约等于
1–2 字
中文 1 token 约等于
×2~3
同一句话，某些小语种 token 数可能是英文的数倍

因为计费、上下文长度、推理速度都按 token 算，分词器对某些语言不友好就意味着这些语言更贵、更慢、可用上下文更短。这也是「分词公平性」近年受关注的原因。

核心瓶颈
  • 多语言不公平：词表偏向英文，小语种和中文被切得更碎。
  • 数字与代码处理别扭：数字被切成奇怪片段，影响算术能力；缩进等空白也敏感。
  • 词表一旦定死就难改：分词器在预训练前就固定，后期几乎无法调整。
  • 探索无分词（byte-level/token-free）：直接吃字节是个方向，但目前效率仍打不过子词。

模型架构：Transformer 与注意力机制

几乎所有现代大模型都建立在 Transformer 之上。理解「注意力」，就理解了大模型为什么这么强。

这一章稍微硬核一点，但只要抓住一个核心概念——注意力（Attention）——其余都是工程细节。

一个直觉：注意力在做什么

处理「它」这个词时，模型需要知道「它」指代前文的哪个东西。注意力机制让每个词去「环顾」句子里所有其他词，按相关程度分配权重，再把相关词的信息加权汇总过来。这就是「任意两词直接互相看到」的含义，也是它能抓住长距离依赖的根本原因。

对每个 token 算出三个向量：
  Query (我在找什么) · Key (我是什么) · Value (我携带的信息)
注意力权重 = softmax(Query · Key / √d)   # 谁和我相关
输出 = Σ 权重 × Value                      # 把相关信息汇总过来

Transformer 的积木

  • 多头注意力：并行开多个「注意力头」，分别关注不同类型的关系（语法、指代、语义……）。
  • 前馈网络（FFN）：每个位置上的小型全连接网络，负责「加工/存储知识」，参数量的大头其实在这里。
  • 残差连接 + 层归一化：让几十上百层能稳定训练、梯度传得动。
  • 位置编码：注意力本身不分先后，需要额外告诉模型「词的顺序」（现代常用 RoPE 旋转位置编码，利于外推长上下文）。

把「注意力 + FFN」这样一层层堆叠几十到上百层，再加大每层宽度，就得到了大模型。

进阶：把注意力想成一场「图书馆检索」

每个 token 同时扮演三个角色：它带着一张查询单 Query（我现在想找什么信息），一个标签 Key（我这里有什么），和内容 Value（我能提供的实际信息）。注意力做的事，就是拿每个词的 Query 去和所有词的 Key 比对相似度，相似度高的「检索命中」，就把对方的 Value 多取一点过来。「它/这本书」之所以能跨越很远对应到正确的指代对象，就是因为它们的 Query 和 Key 在训练中学会了互相匹配。

多头就是同时开好几套这样的检索系统，每套关注不同侧面：有的头专门追踪语法主谓关系，有的追踪指代，有的追踪主题。最后把各头结果拼起来，信息维度就丰富了。

而真正「存知识」的地方，多数人会忽略：是每层里的前馈网络 FFN。注意力负责「在上下文里搬运信息」，FFN 负责「加工并存储从训练中学到的世界知识」，参数量的大头其实在这。MoE 之所以拆的是 FFN，正因为知识主要存在这里——把它做成「很多专家、每次只用几个」，就能在不增加单次计算的前提下塞进更多知识。

解码器架构：现代生成式大模型的标配

GPT 系采用 仅解码器（decoder-only）+ 自回归：从左到右，每次预测下一个 token，预测出的又作为输入继续预测。这种简单到极致的目标，配合规模，催生了惊人的通用能力。

过去 vs 现在：架构的演进

之前
  • RNN/LSTM 顺序处理，难并行
  • 编码器-解码器结构为翻译而生
  • 稠密模型：每个 token 都激活全部参数
现在
  • Decoder-only Transformer 一统天下
  • RoPE 位置编码、RMSNorm、SwiGLU 等成为标配优化
  • MoE 混合专家：每个 token 只激活一小部分「专家」参数，用更少计算撑起更大模型
  • 长上下文优化（GQA 分组注意力、滑窗、稀疏注意力）
MoE：用「稀疏激活」破解规模诅咒

普通稠密模型，参数翻倍计算也翻倍。MoE 把 FFN 拆成很多「专家」，每个 token 只走其中 1～2 个，于是总参数可以很大（强），但单次实际计算很小（快）。代价是显存占用高、训练时专家负载难均衡。

核心瓶颈
  • 注意力的二次复杂度：计算量随序列长度平方增长，是长上下文最大的拦路虎（FlashAttention、稀疏/线性注意力都是在攻这个点）。
  • 长上下文外推：训练时上下文短，推理想要更长就会性能衰减。
  • 架构创新边际收益递减：多数变体相对原始 Transformer 提升有限，规模与数据仍是主导因素。

预训练：在整个互联网上「猜下一个词」

这是最烧钱、最核心的阶段。一个看似简单的任务——预测下一个 token——把语言规律和世界知识压进了上千亿个参数里。

预训练的目标简单到不可思议：给定前文，预测下一个 token。就这一句话，重复几万亿次，模型就从随机数变成了「懂世界」的基座。

训练循环到底在干什么

① 前向读入一段文本，预测每个位置的下一个 token
→
② 算损失预测分布和真实下一个词的差距（交叉熵）
→
③ 反向计算每个参数该往哪调（梯度）
→
④ 更新优化器（AdamW）微调全部参数
↺ 重复万亿次

关键在于：这是自监督——「正确答案」就是文本里本来的下一个词，不需要任何人工标注。这正是它能吃下整个互联网的原因，也是相对「之前」最大的解放。

为什么「猜词」能学到知识

要准确预测「中国的首都是__」后面的词，模型必须「知道」答案是北京。要续写一段代码，它得理解语法和逻辑。压缩即理解：为了把海量文本压进有限参数里准确预测，模型被迫学会了语法、事实、推理、风格等内在规律。

过去 vs 现在

之前（预训练 + 任务微调时代）
  • 预训练只是「起点」，真正干活前要在每个任务的标注数据上再微调
  • BERT 用「完形填空」（掩码预测）做双向理解
  • 模型规模有限，单卡或少量卡即可
现在（通用基座时代）
  • 预训练出的 基座模型本身就极强，靠提示词即可零样本干活
  • 统一用「预测下一个词」的自回归目标
  • 千卡万卡集群、数月连续训练，训练 token 量达数万亿到十几万亿
  • 训练后期混入更高质量数据、退火学习率（annealing）压榨最后性能
进阶：训练循环里到底发生了什么

① 几乎不重复看数据。和小模型时代「同一份数据反复训几十轮（epoch）」不同，现代大模型数据极多，通常只把语料过一遍（约 1 个 epoch）。所以主要矛盾不是「过拟合」（背书），而是「欠拟合」——算力和数据是否足够把规律学透。

② 优化器为什么吃掉那么多显存。主流优化器 AdamW 会为每个参数额外保存两份状态（梯度的一阶、二阶动量）。所以训练时显存占用大致是「参数 ×（自身 + 梯度 + 两份动量）」，是参数本身的好几倍——这正是必须把训练切到上千张卡的直接原因（见下一章）。

③ 学习率不是常数。训练初期用 warmup 把学习率从 0 慢慢升上去（防止一上来就崩），中后期再缓慢衰减（decay/annealing）把模型「磨细」。学习率调度是稳定性与最终效果的关键旋钮之一。

④ 「涌现能力」。有些能力（如多步算术、按指令执行）在模型小的时候几乎为零，规模越过某个门槛后突然出现，这被称为涌现（emergence）。它是「做大」如此诱人的原因之一；不过也有研究指出，部分「涌现」是评测指标选择造成的错觉，学界仍有争论。

产出：Base 模型

预训练结束得到的叫 基座 / Base 模型。它博学、能续写，但不一定听话——你问它问题，它可能续写出更多类似的问题而不是回答。把它变成「助手」，是后面 SFT 和对齐阶段的事。

核心瓶颈
  • 算力与成本：训练一个前沿模型动辄数千万美元、上万张 GPU、数月不间断，是整条流水线最贵的一环。
  • 训练稳定性：规模一大，loss 容易突然飙升（spike）、梯度爆炸，半路崩溃损失惨重，需要大量监控和容错。
  • 数据见顶：可用的高质量 token 接近耗尽（见数据章）。
  • 能耗与硬件供应：电力、散热、GPU 供货都成了现实约束。

扩展法则与算力基础设施

为什么大家拼命把模型做大?因为「Scaling Law」告诉我们：投入越多，效果可预测地越好。但把上万张卡协同起来,本身就是一座工程高山。

扩展法则（Scaling Laws）：大模型时代的指南针

研究发现，模型的能力（用损失衡量）与三个量呈平滑、可预测的幂律关系：模型参数量、训练数据量、计算量。换句话说——只要按比例同时加大模型、数据和算力，效果就会稳定提升。这是大模型敢于「重注做大」的理论底气：可以用小规模实验外推、预测大模型的效果，再决定要不要投钱。

Chinchilla 的关键修正

早期大家拼命堆参数，数据喂得不够。2022 年 Chinchilla 指出：参数和数据应当大致同比增长。很多大模型其实「训练不充分」——同样的算力，做小一点但喂更多数据反而更强。这直接改变了后续模型的配方，也催生了「小而精」的高性价比模型潮流。

进阶：把「算力」换算成钱、卡和时间

业界有个好用的估算公式：训练一个模型需要的总计算量 ≈ 6 × 参数量 × 训练 token 数（单位是浮点运算 FLOPs）。这个简单的乘法解释了一切：参数翻倍或数据翻倍，算力（也就是钱和时间）都翻倍。

举例感受量级：一个千亿参数、用十几万亿 token 训练的模型，总算力达 10²⁵ FLOPs 级别，对应上万张顶级 GPU 连续跑数月，电费 + 折旧动辄数千万美元。这就是为什么「要不要做这个模型」是 CEO 级决策，也是为什么能先用小实验把 Scaling 曲线拟合出来、预测大模型效果如此有价值——它把一次豪赌变成了可计算的投资。

Chinchilla 的经验法则可以记成：每个参数大约配 20 个训练 token最划算。但这是「训练最省」的配比；如果模型要长期上线服务亿万次，人们会故意「过度训练」小模型（喂远超 20 倍的数据），因为小模型推理更便宜，长期总成本更低——这就是近年「小而强」模型流行的经济学原因。

为什么一张卡远远不够

一个千亿参数模型，光是参数 + 梯度 + 优化器状态就需要 TB 级显存，远超单卡（几十 GB）。所以必须把训练切开，分布到成百上千张卡上协同。这就是「分布式并行」。

四种并行方式（现代训练的拼图）

并行方式切什么解决什么
数据并行 (DP)每张卡放完整模型，喂不同的数据批次加快吞吐；最基础
张量并行 (TP)把单层的大矩阵切成几块，分到多卡同时算单层放不下时
流水线并行 (PP)把不同的层分到不同卡，像流水线接力模型层数太多时
专家并行 (EP)把 MoE 的不同专家分到不同卡MoE 模型专用

外加 ZeRO / FSDP 技术：把优化器状态、梯度、参数也分片到各卡，进一步省显存。前沿训练通常是这几种方式叠加组合（3D / 4D 并行）。

省显存、提速度的关键工程

  • 混合精度（FP16/BF16，乃至 FP8）：用更低精度算，省一半以上显存和带宽。
  • 梯度检查点（activation checkpointing）：用时间换显存，反向时重算激活而非全存。
  • FlashAttention：重写注意力计算，避免显存搬运瓶颈，大幅提速并支持更长序列。
  • 高速互联：NVLink / InfiniBand，让卡间通信不拖后腿。
核心瓶颈
  • 通信墙：卡越多，同步梯度/激活的通信开销越大，常常成为瓶颈而非算力本身。
  • 显存墙：显存增长远跟不上模型增长，逼出了各种分片和重算技巧。
  • 利用率（MFU）：再好的集群，实际算力利用率常只有 30%～50%，剩下都耗在等待和通信上。
  • 故障率：上万张卡跑数月，硬件几乎必然出故障，需要 checkpoint 和自动恢复，否则前功尽弃。

监督微调（SFT）：教模型「听懂人话」

基座模型博学却不听话。SFT 用成千上万条「指令 → 理想回答」的示范，把它从「续写机器」变成「能干活的助手」。

预训练出的基座模型，你问它「帮我写封请假邮件」，它可能续写成「……以及如何写辞职信、如何写道歉信」。它学会了语言，却没学会「应答」。监督微调（Supervised Fine-Tuning, SFT），也叫指令微调（Instruction Tuning），就是来补这一课的。

做法：示范学习

准备大量「指令 + 理想回答」的成对数据，让模型在这些示范上继续用「预测下一个词」的方式训练——但这次只学着模仿回答部分。本质还是预训练那套机制，只是数据换成了高质量的对话/任务示范。

指令: 把这句话翻译成英文：今天天气很好。
理想回答: The weather is nice today.

指令: 用一句话解释什么是黑洞。
理想回答: 黑洞是引力极强、连光都无法逃逸的天体区域。

为什么少量数据就能见效

关键洞察：模型在预训练时其实已经具备这些能力，SFT 不是在教新知识，而是在激活并规范输出格式——告诉它「面对指令时，应该用这种方式回应」。所以几千到几十万条高质量数据往往就够，质量远比数量重要（著名的「LIMA：少即是多」实验说明了这点）。

过去 vs 现在

之前（任务微调）
  • 为每个任务单独微调一个模型（一个分类、一个翻译……）
  • 需要该任务的大量标注数据
  • 模型不通用，换任务就得重训
现在（指令微调）
  • 把各种任务统一成「指令」格式混在一起训一个通用模型
  • 训练出的模型能泛化到没见过的新指令
  • 大量使用合成/蒸馏数据：用更强的模型生成示范来教较弱模型
  • 覆盖多轮对话、工具调用、代码、特定格式等

高效微调：LoRA 等

全参数微调很贵。LoRA / QLoRA 等「参数高效微调（PEFT）」只训练少量新增的低秩矩阵，冻结原模型，用极小的成本就能定制模型——这让中小团队也能微调大模型，是开源生态繁荣的重要推手。

核心瓶颈
  • 数据质量天花板：SFT 效果几乎完全取决于示范数据的质量，而高质量标注昂贵且难规模化。
  • 模仿的局限：SFT 只能模仿示范，学不会「什么是更好的回答」——它没有「比较」的概念，遇到示范没覆盖的情况就抓瞎。这正是需要下一阶段 RLHF 的根本原因。
  • 幻觉风险：如果示范里要求模型回答它其实不知道的问题，反而会「教会它自信地编造」。
  • 灾难性遗忘：微调过头会损害预训练学到的通用能力。

对齐：用人类偏好把模型「调教」得有用又安全

SFT 只会模仿,不懂好坏。对齐阶段让模型学会「人更喜欢哪个回答」,这是 ChatGPT 体验的真正秘密。

SFT 之后模型已经能应答，但还不够「贴心」：回答可能啰嗦、回避、不安全，或者在多个合理回答中选了不那么好的那个。对齐（Alignment）的目标，是让模型符合人类偏好——通常概括为 3H：Helpful（有用）、Honest（诚实）、Harmless（无害）。

RLHF：基于人类反馈的强化学习

这是 InstructGPT / ChatGPT 的核心方法，分三步：

① 收集偏好同一问题让模型生成多个回答，人类标注「哪个更好」
→
② 训练奖励模型 (RM)用偏好数据训练一个模型，学会给任意回答打分
→
③ 强化学习用 RM 当「裁判」，通过 PPO 等算法优化语言模型，让它倾向于拿高分的回答
为什么是「比较」而不是「打分」

让人直接给回答打绝对分（比如 7.5 分）很不稳定，不同人尺度不一。但让人判断「A 和 B 哪个更好」要可靠得多。RLHF 巧妙地把「难以定义的好坏」转化成了「成对比较」，再让奖励模型把这种偏好泛化开。这解决了 SFT「只会模仿、不懂优劣」的根本短板。

一个关键约束：别跑偏（KL 惩罚）

RL 优化时如果一味追求高奖励，模型会找到奖励模型的漏洞、输出怪话（奖励黑客 reward hacking）。所以会加一个约束：不许偏离原 SFT 模型太远（KL 散度惩罚），在「讨好裁判」和「保持正常」之间平衡。

进阶：PPO 阶段为什么要同时跑「四个模型」

经典 RLHF 用 PPO 时，显存里常常同时装着四个模型，理解它们各自的角色，就理解了为什么这一步又贵又难调：

  • 策略模型（Policy）：正在被训练、要变强的那个语言模型本体。
  • 参考模型（Reference）：训练开始时策略模型的「冻结快照」，用来算 KL——衡量「现在偏离初始有多远」，防止越训越歪。
  • 奖励模型（Reward）：前面用人类偏好训出来的「裁判」，给每个回答打分。
  • 价值模型（Value/Critic）：估计「当前局面预期能拿多少分」，帮助判断每一步是赚是亏（降低训练噪声）。

四个模型一起跑，显存和工程复杂度都翻倍，还容易不稳定——这正是后来大家想方设法简化它的动机。

进阶：DPO 的巧妙之处

DPO 的洞察是：上面那套「训奖励模型 + 跑 RL」在数学上其实可以化简成一个直接的损失函数。它不再显式训练裁判，而是直接告诉模型——「对于这一对回答，提高被偏好那条的相对概率、压低被嫌弃那条的」，同时用参考模型约束别跑太远。一步到位，省掉了奖励模型和强化学习的全部复杂度，所以稳定、便宜、好复现，成了开源社区的默认选择。代价是它通常需要现成的成对偏好数据，灵活性和上限不如在线 RL。

过去 vs 现在：从 PPO 到 DPO

经典 RLHF（PPO）
  • 要单独训一个奖励模型 + 跑复杂的强化学习
  • 同时要加载 4 个模型，工程复杂、不稳定、调参难
  • 效果好但门槛高
DPO 及新方法
  • DPO（直接偏好优化）：数学上证明可以跳过奖励模型和 RL，直接用偏好数据微调，简单稳定
  • 衍生出 IPO、KTO、ORPO、SimPO 等一系列变体
  • RLAIF / Constitutional AI：用 AI 代替部分人类标注（按一套「宪法」原则让模型自我批评修正），大幅降低标注成本
DPO vs PPO 怎么选

DPO 简单、便宜、稳定，是开源社区主流；在线 RL（PPO 类）上限更高、更灵活，前沿实验室仍在用，尤其在下一章的「推理强化」里 RL 重新成为主角。两者并存，各有适用场景。

核心瓶颈
  • 奖励黑客：模型钻奖励模型的空子，比如学会「说得长、说得自信」来骗高分，而非真的更好。
  • 人类偏好不一致、有偏：标注者意见相左、带有主观和文化偏见，且偏好会被「写得讨喜」误导（谄媚 sycophancy）。
  • 对齐税（alignment tax）：对齐可能轻微损害某些原始能力，需要权衡。
  • 众包标注昂贵且难扩展，催生了用 AI 反馈替代的探索。
  • 难以对齐「我们说不清的目标」：当任务超出人类轻松评判的范围，如何对齐成为开放难题（可扩展监督）。

推理模型：教模型「先想清楚再回答」

2024–2025 最大的范式变化。与其让模型脱口而出，不如让它生成长篇思考过程,再用「可验证的奖励」强化正确的推理。

传统对话模型是「脱口而出」——看到问题立刻给答案。但复杂的数学题、编程题、逻辑推理，人类也要打草稿、试错、回溯。推理模型（Reasoning Models）正是让模型学会这件事，代表是 o1 风格模型和开源的 R1 类工作。这是继 RLHF 之后又一次重要范式跃迁。

核心思想：思维链 + 可验证奖励

两个支柱：

  • 长思维链（long Chain-of-Thought）：训练模型在给出最终答案前，先生成一长段「内部思考」——分解问题、尝试、自我检查、纠错。
  • RLVR（基于可验证奖励的强化学习）：在数学、代码这类答案可以自动判对错的领域，奖励信号不再依赖易被钻空子的奖励模型，而是用客观验证器（答案是否正确、代码是否通过测试）。这让 RL 信号干净可靠，可以大规模自动进行。
为什么这是个突破

RLHF 的奖励来自「人类觉得好不好」，容易被谄媚和奖励黑客污染。而「这道数学题答对没有」是客观、可自动验证、不会撒谎的。把 RL 建立在这种可靠信号上，模型可以通过海量自我尝试、自我纠错真正学会推理，而不只是学会「显得在推理」。某些工作甚至显示，仅靠 RL（几乎不用人工示范）就能涌现出反思、回溯等高级推理行为。

进阶：强化学习到底怎么「教会」模型推理

核心直觉非常朴素，叫「试错 + 强化」：对同一道题，让模型采样出很多条不同的解法，用验证器判对错，然后——把通向正确答案的那些推理路径的概率调高，把走向错误的调低。重复千万次，模型就逐渐「长出」分解问题、自我检查、回溯纠错这些行为，因为这些行为统计上更常通向正确答案。

这里有个关键升级叫 GRPO（也是 R1 类工作采用的方法）：它砍掉了上一章 PPO 里那个昂贵的价值模型，改用「一组答案里，谁比这组的平均分高就上调、低就下调」的相对比较来估计「这条路好不好」。少跑一个大模型，训练更省更稳，是推理强化能规模化的工程关键之一。

为什么可验证奖励（RLVR）是灵魂：RL 最怕奖励信号被钻空子。「这道数学题答案对不对、这段代码测试过不过」是客观、自动、不会撒谎的信号，于是模型无法靠「显得聪明」骗分，只能真的把题做对。干净的奖励，才撑得起千万次的自我训练。

测试时计算（Test-time / Inference-time Scaling）

这带来一条新的扩展维度：过去靠「训练时」堆参数和数据；现在发现，让模型在回答时多「想」一会儿（生成更长的思考、或采样多条路径再择优），也能稳定提升正确率。也就是说——愿意多花推理算力，就能换来更高质量的答案。这把「扩展法则」从训练侧延伸到了推理侧。

过去 vs 现在

之前
  • 模型直接输出答案，复杂推理靠提示词里写「Let's think step by step」临时诱导
  • 对齐主要追求「说得好听、有用、安全」
  • 能力上限主要由训练时规模决定
现在
  • 把「深度思考」内化进权重，模型自发地长篇推理
  • 用 RLVR + 客观验证器大规模训练推理能力
  • 新增「测试时计算」维度：推理越久越准
  • 催生 GRPO 等更高效的 RL 算法（去掉价值网络，降低成本）
核心瓶颈
  • 只在「可验证」领域好用：数学、代码容易自动判分，但写作、开放对话、共情这类没有标准答案的任务难以构造可靠奖励。
  • 推理成本暴涨：一条长思维链可能消耗成千上万 token，又慢又贵；如何「该想多久想多久」（自适应思考预算）是难题。
  • 过度思考：简单问题也长篇大论，浪费算力甚至把对的想错。
  • 思考过程未必忠实：模型展示的「思路」不一定是它真实的决策依据，给可解释性和安全监督带来挑战。
  • RL 训练仍不稳定、对配方敏感，复现门槛高。

推理部署：把模型跑得又快又便宜

训练只做一次,推理却要服务亿万次请求。模型上线后,真正决定成本的是推理效率。

一个模型训练好只是开始。真正面对用户时，是推理（inference）在反复运行。对大规模服务来说，推理的总成本往往远超训练，所以「如何让模型又快又省又稳」是一门独立的硬功夫。

自回归推理为什么慢

生成是一个 token 一个 token挤出来的，每生成一个都要把整个模型跑一遍，且必须等前一个出来才能算下一个——天然无法并行。更要命的是，它是访存瓶颈（memory-bound）：瓶颈不在算力，而在把上百 GB 的权重反复从显存搬进计算单元。

关键优化技术

技术解决什么原理
KV Cache避免重复计算缓存已生成 token 的 Key/Value，新 token 只需算自己，不必重算全部历史
量化 (Quantization)省显存、提速把权重从 16 位压到 8 位、4 位甚至更低，模型变小、搬运更快，精度仅小幅损失
连续批处理提高吞吐动态地把多个请求拼在一起处理，让 GPU 不空转（vLLM 等框架的核心）
PagedAttention省显存碎片像操作系统分页一样管理 KV Cache，显存利用率大增
投机解码提速用小模型先快速猜几个 token，大模型一次性批量验证，验证比逐个生成快
蒸馏 (Distillation)降本用大模型当老师，训练一个小模型逼近其表现，上线跑小模型

过去 vs 现在

之前
  • 模型小，单卡就能跑，推理优化不太被重视
  • 简单地一条条请求处理
现在
  • 专门的推理引擎（vLLM、TensorRT-LLM、SGLang 等）成为基础设施
  • KV Cache 管理、连续批处理、量化成为标配
  • MoE 模型推理：总参数大但单次激活少，专门优化专家路由与显存
  • 推理模型（长思维链）让单次请求 token 暴增，推理优化更加关键
核心瓶颈
  • 显存带宽墙：推理受限于搬运权重的带宽，而非算力，这是根本约束。
  • 长上下文 / 长思考的 KV Cache 膨胀：上下文越长，KV Cache 占的显存越多，可能比模型本身还大。
  • 延迟 vs 吞吐 vs 成本的三角权衡：要快（低延迟）、要省（高吞吐）、要便宜，三者难以兼得。
  • 量化的精度损失：压得太狠会掉点，尤其伤害推理类复杂任务。

评估：怎么知道模型到底行不行

训练每一步都要回答一个问题:这个模型变好了吗?但评估大模型,远比想象中困难和不可靠。

评估贯穿整条流水线：选数据配比要评、预训练中途要评、对齐前后要评、上线前更要评。但「衡量一个通用智能体到底有多强」本身就是个未解难题。

评估的几个层次

  • 基准测试（Benchmarks）：标准化考试。如 MMLU（多学科知识）、GSM8K / MATH（数学）、HumanEval（代码）、知识问答、常识推理等，能给出可比较的分数。
  • 人工评测：让人对回答打分或两两比较，更贴近真实体验，但慢、贵、主观。
  • LLM-as-a-Judge：用强模型当裁判自动评分，便宜可扩展，是现在的主流之一，但裁判本身有偏见。
  • 竞技场（Arena）：让真实用户盲选两个模型的回答，用类似象棋 Elo 的机制排名，反映真实偏好。

过去 vs 现在

之前
  • 任务单一，指标明确（翻译看 BLEU、分类看准确率）
  • 一个测试集就能说明问题
现在
  • 模型是通用的，没有单一指标能概括，要一整套基准 + 人评 + 竞技场组合
  • 越来越重视真实任务、智能体（agent）能力、长上下文、安全性、多模态
  • 不断推出更难的基准（旧的很快被刷满），如研究生级难题、前沿数学
核心瓶颈
  • 数据污染：基准题目泄漏进训练数据，分数虚高，无法反映真实能力（评估的头号公敌）。
  • 基准饱和 / 应试：模型针对刷榜优化，高分不等于真本事；好的基准很快被刷爆。
  • 裁判偏见：LLM 裁判偏爱更长、格式更漂亮、和自己风格相近的回答。
  • 开放任务无标准答案：创意写作、对话、共情等难以客观打分。
  • 能力越强越难评：当模型在某领域超过普通标注者，人类已难以判断对错（可扩展监督难题）。

范式正在往哪走

  • 从「训练时扩展」到「推理时扩展」：愿意多花推理算力换正确率，是 2024 以来最重要的新轴。
  • 从「会聊天」到「会做事」：智能体（agent）方向——模型调用工具、写代码、多步规划、与环境交互，RL 在真实环境里训练成为热点。
  • 数据效率优先：在数据见顶的前提下，「用更少数据学更多」比「无脑堆量」更重要。
  • 多模态融合：文本、图像、音频、视频统一进同一模型。
  • 小模型 + 蒸馏 + 端侧：高性价比小模型让能力下沉到手机和边缘设备。
  • 对齐与安全的长期挑战：当模型能力逼近甚至超过人类评判水平，「可扩展监督」「忠实可解释」成为核心研究方向。

① 大模型 = 预训练学知识 + 后训练学做人 + 推理强化学思考。
② 每一次范式跃迁，都源于上一代瓶颈被打破：规则→统计→RNN→Transformer→RLHF→推理强化。
③ 当下三道墙——数据、算力、评估——既是约束，也指向下一个突破口。

顶级模型公司在如何竞争？训练方式还在变吗？

前面讲的是「怎么训练一个模型」。这一章拉远镜头：站在 2026 年，头部公司在比什么、主要做什么，以及在模型周以周计迭代的今天,训练范式本身正在怎样改变。

他们到底在比什么

竞争早已不只是「谁的模型更聪明」这一个维度，而是分化成好几条战线：

竞争维度在比什么典型表现
绝对智能上限最难的推理、数学、科研级任务谁更强各家轮流登顶各类高难基准

关键趋势：从「一家通吃」到「专业分工」

2026 年最明显的特征是没有任何单一模型在所有品类都最好。编程、推理、写作、多模态、工具调用各有领跑者。竞争从「一个全能冠军」转向「多个单项冠军 + 一个生态最大的全能选手」的格局。

头部公司主要在做的几件事

  • 把「推理/思考」做深：用强化学习训练模型先长篇思考再作答，并能动态分配思考算力（简单题快答、难题多想）。
  • 押注智能体（Agent）：让模型不只是对话，而是能连续操作工具、写代码、跑任务、自主工作几小时——这被普遍视为下一个主战场。
  • 统一/路由架构：把「快模型」和「深思考模型」合成一个系统，用内部路由器按请求难度实时选择，兼顾体验与成本。
  • 死磕推理成本：模型再强，跑得贵就没人用。量化、蒸馏、MoE、推理引擎优化成为产品竞争力的一部分。
  • 抢生态位：绑定云平台、IDE、浏览器、办公软件，把模型变成「随处可用的能力」。
  • 安全与对齐：能力越强，可控性、可解释、防滥用越成为差异化和合规门槛。

核心问题：迭代这么快，训练方式本身在变吗？

答案是正在发生明显转移。前几章描述的流水线骨架没变，但「投入的重心」和「能力的来源」已经和两三年前不同：

早期范式（GPT-3/4 时代）
  • 能力主要来自预训练时堆参数、堆数据、堆算力（Scaling Law 主导）
  • 后训练（SFT + RLHF）是「锦上添花」的对齐环节
  • 模型「脱口而出」，复杂推理靠提示词临时诱导
  • 更大 = 更强，几乎是唯一信仰
当前范式（2025–2026）
  • 预训练的边际收益递减（叠加优质数据见顶），单纯做大不再稳赚
  • 重心转向后训练与强化学习：用 RL（尤其可验证奖励 RLVR）大规模训练推理能力，成为拉开差距的关键
  • 新增「测试时计算」这条扩展轴——让模型回答时多想、多采样、自我验证，花推理算力换正确率
  • 转向智能体式训练：在真实/模拟环境里用 RL 训练模型完成多步任务、使用工具
  • 数据从「堆量」转向「质量 + 合成数据 + 数据效率」
几条正在成形的新方向
  • 把价值函数/验证器请回 RL：GRPO 等方法曾丢掉价值网络，新工作又把「推理器 + 验证器」联合训练，以更好支撑测试时的搜索与自我核查。
  • 测试时训练 / 自我改进：让模型在面对新任务时临时用自己的「多数投票」当伪标签做轻量 RL，或用可训练的「记忆」组件提升多轮表现。
  • 自适应思考预算：训练模型自己判断「该想多久」，避免简单题过度思考、难题想得不够。
这一阶段的核心瓶颈
  • 预训练红利见顶：再做大的回报不如从前，必须从别处（RL、推理、数据效率）找增量。
  • RL 的可扩展性：RLVR 在数学/代码好用，但如何把可靠奖励推广到开放任务、智能体长流程，仍是难题。
  • 推理成本与产品体验的拉扯：长思考、长上下文、智能体都极耗算力，「又强又快又便宜」三角依旧难破。
  • 评估与可信：能力越强越难判断真伪（见评估章），叠加安全合规压力，成为头部公司的长期负担。
  • 同质化与护城河：开放权重快速追赶、彼此蒸馏，技术差距容易被抹平，竞争越来越落到生态、成本和工程执行力。
一句话收尾

头部公司的竞争已从「谁的模型更大」演变为「谁更会用强化学习教模型思考、谁的智能体更能干活、谁能把这一切跑得更便宜并嵌进生态」。训练范式的重心，正从预训练时扩展滑向后训练与推理时扩展——这也呼应了第 10 章「推理强化」的主线。

迭代速度之谜：为什么有人快，有人慢

顶级公司为何能以「周」为单位刷新模型？而中国头部公司又面临什么不同的处境？这一章把「迭代速度」拆开，看清背后的飞轮与约束。

时效与立场说明

本章是产业分析，整合自 2026 年中的公开报道与行业数据，存在不确定性与不同观点；目的是讲清「速度差异背后的结构性原因」，而非评判优劣或选边。具体数字会很快变化，请关注逻辑。文末附来源。

三年间，前沿模型的更新频率压缩了三倍以上。这种速度本身已经成为一种竞争武器——谁能更快把改进推给用户、更快收集反馈，谁就能滚得更快。问题来了：这种速度从哪来？

问题一：顶级公司为什么能这么快？

关键要打破一个误解：他们并不是每周从头训练一个新基座模型（那需要三四个月、上千万美元，不可能每周来一次）。真正的秘密，是一套环环相扣的迭代飞轮。

飞轮的核心：预训练一次，后训练很多次

最贵最慢的预训练（基座）只做少数几次。一旦有了强基座，后续大量的版本更新发生在后训练阶段——SFT、对齐、推理强化。按 2025 年的数据，后训练大约只占整体算力的 15%～25%，相对便宜得多。这意味着：同一个基座，可以快速衍生出一连串后训练版本（更会推理的、更会用工具的、更安全的）。绝大多数「这周的新模型」其实是后训练层面的迭代，而非重新预训练。这就是高频发布的物理基础。

支撑飞轮高速转动的五个轮轴

要素为什么能加速迭代
常备评测流水线建好一整套自动化评测（占 AI 总开销约 3%～5%），任何改动几小时内就知道「变好还是变坏」，于是敢于高频发布。能快速、可信地度量，是敢快的前提。
基础设施复用训练框架、数据管线、RL 环境、部署引擎一次建好反复用。新模型是「在成熟流水线上跑一遍」，而不是从零搭。
资本 + 算力飞轮收入→融资→买更多卡→训更强模型→更多收入。美国头部实验室的资本量级是中国同行的 30 倍以上（如有报道称的千亿级算力投入），算力不再是稀缺品，可以同时跑很多实验。
蒸馏 + 合成数据用最强的「教师模型」快速生成训练数据、或蒸馏出更小更快的版本，免去重新积累数据的漫长过程。
人才密度 + 产研一体顶尖研究者高度集中，研究与产品紧耦合，线上反馈直接喂回训练，形成「真实使用 → 数据 → 更强模型」的闭环。
一个反直觉的现实：现在的瓶颈常常是「卡和电」，不是「会不会做」

2026 年的新约束是：用最新一代芯片训出的模型，要等到有足够的推理部署容量（GPU + 电力 + 数据中心）才能上线。也就是说，限制发布的越来越是物理产能，而非算法能力。这反过来解释了为什么「算力 + 资本 + 电力」成了头部公司军备竞赛的真正战场。

问题二：中国头部公司是不是「没那么快」？

先校正前提：在「效率赛道」上，中国头部模型其实迭代得又快又省

事实上，DeepSeek、阿里 Qwen、月之暗面 Kimi、智谱 GLM 等近年的迭代节奏相当快，而且以「高性价比」著称：在编程、数学等部分基准上达到甚至领先国际水平，开源影响力很大。最有代表性的是，有报道称 DeepSeek 的 R1 训练成本约 600 万美元，相比 GPT-4 约 1 亿美元量级低了一个数量级，靠的正是 MoE「只激活一小部分参数」等效率优化。所以「中国公司迭代慢」并不准确——更准确的说法是：在「绝对前沿」（最大规模、最尖端硬件、最大推理集群）这条特定赛道上，节奏受到结构性约束，而且战略重心本就不同。

真正的约束在哪：四个结构性因素

① 算力 / 芯片

美国出口管制切断了最先进 GPU（H100/A100 及其降配版 H800/A800 均受限），中国实验室主要依赖受限的 H20，并被引导转向华为昇腾等国产芯片。这直接卡住了最大规模的预训练长跑和大规模推理部署——不是不会做，而是「同时能开多少张卡、训多久」被限制了。

② 资本规模

有统计称 2024–2026 年中国前沿实验室公开融资合计不足百亿美元，而美国同行高出 数十倍；叠加资本管制、CFIUS/出境投资限制，国际资本难以进入。最烧钱的「超大训练 run + 海量买卡」因此更难承受。

③ 人才结构

顶尖研究者并不缺，但有分析指出大批量工程人才与落地工程能力相对紧张；同时人才流动与外流受到更多限制，影响规模化扩张的速度。

④ 战略与生态差异

中国路线更强调效率、开源、落地（AI+ 各行业），是「多条赛道并进」而非单押「绝对前沿 AGI」；同时在全球云分发、海外产品生态上的触达较弱，商业化与监管环境也不同。重心不同，自然不会在「刷最大模型」上和对手比谁更快。

所以更精确的结论是

中国头部公司选择了一条「用效率与开源对冲算力与资本劣势」的路线：把每一张卡、每一块钱的产出做到极致（MoE 稀疏激活、蒸馏、极致工程优化）。在效率与性价比维度，迭代很快；但在「绝对前沿军备竞赛」（最大模型、最长训练、最大推理集群）维度，节奏被芯片与资本约束、战略上也不强求。换句话说，「快慢」取决于你拿哪条赛道做标尺。

维度美国头部实验室中国头部实验室（典型）
核心打法资本 + 算力堆绝对前沿，产研闭环高频发布效率优先、开源放大影响、深耕落地

把训练范式的变化和迭代速度联系起来

回到全文主线：正因为能力来源正从「预训练时扩展」转向「后训练 + 强化学习 + 推理时扩展」（第 10、14 章），迭代才可能这么快——后训练便宜、可快速试错、效果可即时评测。这也给了算力受限者一条出路：与其在预训练上和对手拼卡，不如在后训练、数据效率、推理优化上拼巧劲。范式的变化，既解释了「为什么快」，也重塑了「谁有机会追上」。

一句话收尾

顶级公司快，是因为「预训练一次、后训练多次」+ 自动评测 + 基建复用 + 资本算力飞轮把迭代变成了流水线；中国头部公司并非整体慢，而是在绝对前沿被芯片与资本约束、战略上转而主打效率与开源——这是一场「算力的豪赌」与「效率的巧劲」之间的竞赛，而非简单的快与慢。
