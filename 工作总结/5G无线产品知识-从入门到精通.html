    CH 01入门

    什么是 5G？

    5G 是第五代移动通信技术（5th Generation）。如果说 4G 解决了"人随时随地上网"的问题，那么 5G 的目标是连接"人、物、产业"——它不只是更快的网，而是一张可以承载工厂、车联网、医疗、城市等行业应用的数字底座。

    一句话理解：4G 改变生活，5G 改变社会。4G 让我们刷视频、点外卖；5G 把网络变成像水电一样的"生产资料"，让工厂、港口、矿山、医院都能跑在无线网络上。

    5G 为什么能这么快？三个根本来源

    速率不是凭空变快的。香农定理告诉我们，容量 ≈ 带宽 × 频谱效率 × 空间复用流数。5G 在这三个乘子上同时发力：

    • 更大的带宽：4G 单载波多为 20MHz，5G 中频单载波可达 100MHz，毫米波更可达数百 MHz——管道直接变粗数倍。
    • 更高的频谱效率：采用 256QAM 高阶调制、LDPC/Polar 新编码、灵活帧结构，让每 Hz 频谱搬运更多比特。
    • 更多的空间流：Massive MIMO 把同一频谱在空间上"复用"出几十路并行数据流，这是 5G 容量跃升的最大贡献者（详见第 4 章）。

    理解 5G 的三层能力体系

    不要把 5G 只理解成"快"。它是一个分层的能力体系：底层是空口与无线（更快、更低时延），中层是架构能力（切片、边缘、云化），上层是行业使能（确定性网络、数据本地化、安全隔离）。商业价值是逐层向上放大的——越往上越值钱。

    关键术语先打个底

    gNodeB / gNB
    5G 的基站，相当于 4G 的 eNodeB。负责无线信号收发与无线资源调度。
    NR (New Radio)
    5G 的全新空口标准（3GPP 定义），规定了无线接口的"语言"与帧结构。
    UE
    User Equipment，用户终端，如手机、CPE、车载模组、工业网关。
    RAN / 核心网
    RAN（无线接入网）负责无线连接；核心网（Core）负责认证、会话、计费、路由。
    SA / NSA
    独立组网（依赖 5G 核心网，能力完整）/ 非独立组网（借用 4G 核心网起步）。
    3GPP / Release
    制定移动通信标准的组织，按 Release 版本演进：R15 首版 5G、R16/R17 增强、R18 起为 5G-A。
    QoS / SLA
    服务质量 / 服务等级协议。5G 用 5QI 标识不同业务流的时延、可靠性、优先级。
    给售前的提醒：客户会反复问"5G 到底比 4G 强多少"。把答案锚定在带宽、频谱效率、空间流这三件事上，比单纯报峰值速率更专业、也更经得起追问。
    CH 02入门

    5G 的三大应用场景

    国际标准（ITU IMT-2020）把 5G 能力浓缩成三个三角顶点。理解这三类场景，是理解"5G 卖给谁、卖什么"的根本。

    eMBB
    增强移动宽带
    大带宽。4K/8K 视频、VR/AR、热点扩容。
    面向：消费者、家庭宽带（FWA）
    URLLC
    超可靠低时延
    毫秒级时延 + 99.999% 可靠。工业控制、车联网、远程医疗。
    面向：行业/工业
    mMTC
    海量机器连接
    超大连接密度、低功耗。智慧城市、抄表、传感网。
    面向：物联网
    售前视角：消费者市场主要靠 eMBB（更快的手机网络、家庭无线宽带）；而真正的增量利润，往往来自 URLLC 与 mMTC 驱动的 行业专网。这一点在第 11 章会反复用到。

    无线网络架构：信号怎么走的

    一通 5G 数据，从终端出发，要经过终端（UE）→ 无线接入网（RAN）→ 承载网（Transport）→ 核心网（5GC）→ 数据网络（互联网/云/园区）。看懂这条端到端链路，是理解整个 5G 系统的地图。

    无线产品主要工作在 gNB 这一段，但方案交付必须把承载与核心网一起考虑，否则能力打不通。

    RAN 的三种部署形态

    📡

    D-RAN 分布式

    BBU（基带）和 AAU（射频）都在基站站点，部署简单、对承载要求低，适合站点分散、新建初期。

    🏢

    C-RAN 集中式

    多站点 BBU 集中到机房并池化，便于小区协同（CoMP）、省机房与电力，但需要大带宽低时延前传。

    🧩

    O-RAN 开放

    CU/DU/RU 接口标准化开放（如 7.2 前传），支持多厂商解耦、白盒化，降低绑定但集成复杂度上升。

    5G 基站的功能拆分：CU / DU / RU

    5G 把传统一体化基站拆成三个功能单元，按协议栈"高层—实时—射频"分层，这是 5G 架构最核心的变化：

    CU（集中单元）
    处理非实时高层协议（RRC、SDAP、PDCP），可云化、可集中部署，一个 CU 管多个 DU。
    DU（分布单元）
    处理实时基带（RLC、MAC、高 PHY），对时延敏感，靠近站点，决定调度与空口性能。
    RU / AAU（射频单元）
    低 PHY + 射频 + 天线，真正发射电磁波。Massive MIMO 场景下与天线一体化为 AAU。

    协议栈是怎么分到 CU/DU/RU 的

    RRC / SDAP / PDCPCU — 连接管理、加密、QoS 映射（非实时）
    RLC / MAC / High-PHYDU — 调度、HARQ、分段重传（实时）
    Low-PHY / RFRU/AAU — 调制、波束赋形、收发（射频）

    CU 与 DU 之间的切分点（Option 2，PDCP/RLC 之间）是业界主流，兼顾时延与集中化收益。

    关键接口与"三传"

    接口 / 段落连接对象说明
    UuUE ↔ gNB无线空口，5G NR 信号传输的核心接口。
    前传 FronthaulDU ↔ RU/AAU带宽大、时延极敏感（百微秒级），常用 eCPRI / O-RAN 7.2。
    中传 Midhaul（F1）CU ↔ DUF1 接口，时延要求中等，可承载在 IP 承载网上。
    回传 Backhaul（NG）gNB ↔ 5GCNG 接口，连接核心网；N2 信令面 / N3 用户面。
    XngNB ↔ gNB基站间接口，用于移动性切换与双连接协同。
    E1CU-CP ↔ CU-UPCU 内部控制面与用户面分离的接口。

    5G 核心网（5GC）：服务化架构 SBA

    5G 核心网与 4G 最大的不同是采用服务化架构（SBA）：把核心网拆成一组可独立部署、按需调用的网络功能（NF），云原生、易扩展，是切片与边缘能力的根基。

    🔐

    AMF

    接入与移动性管理：注册、鉴权、移动性、连接管理。

    🔀

    SMF

    会话管理：建立/管理 PDU 会话，控制 UPF 转发。

    🚄

    UPF

    用户面转发：数据路由与转发，可下沉到边缘实现低时延。

    🗂️

    UDM / AUSF

    统一数据管理与鉴权服务，存储签约与安全信息。

    🧭

    PCF

    策略控制：QoS、计费与切片策略下发。

    🔎

    NRF / NSSF

    网络功能注册发现 / 切片选择，支撑动态编排。

    架构主线（务必记住）：5G 通过两件事释放价值——RAN 侧把基站拆成 CU/DU/RU 实现云化与灵活部署，核心网侧用 SBA + UPF 下沉实现切片与边缘计算。控制面与用户面分离（CUPS）+ UPF 下沉，是低时延与数据本地化的物理基础，也是行业专网的技术底座。
    CH 04进阶

    核心关键技术

    这些是 5G 性能背后的"硬功夫"，也是售前讲解差异化卖点时绕不开的几张牌。

    🎯

    Massive MIMO 大规模天线

    64T64R 甚至更多天线阵列，让一个基站同时服务更多用户，大幅提升容量与覆盖。

    🔦

    Beamforming 波束赋形

    把信号像手电筒一样精准"打"向用户，而非全向广播，提升信号强度、减少干扰。

    🍰

    Network Slicing 网络切片

    在一张物理网络上切出多张逻辑专网，每张有独立 SLA。行业变现的关键能力。

    ⚡

    MEC 边缘计算

    把算力下沉到网络边缘，数据就近处理，时延更低、数据不出园区，工业刚需。

    📐

    灵活帧结构 / SCS

    可变子载波间隔，兼顾大带宽与低时延，适配不同频段与场景。

    🌱

    绿色节能

    智能关断、符号关断、深度休眠，按业务量动态省电，直接降 OPEX。

    Massive MIMO 为什么是 5G 的"招牌"

    4G 基站通常是 2T2R 或 4T4R，而 5G 主力 AAU 是 64T64R。天线越多，越能做精细的波束赋形和多用户复用——同样的频谱，能服务的用户和承载的流量成倍增长。这是 5G 在拥挤城区"容量碾压"4G 的根本原因。

    • TxR 通道数（如 64T64R）：收发通道越多，波束越窄越精准，空间复用能力越强，但功耗、重量、成本也越高。
    • SU-MIMO 与 MU-MIMO：SU 给单用户叠加多个数据流（提单用户速率）；MU 用空分把同一时频资源同时分给多个用户（提小区容量）。Massive MIMO 的价值主要在 MU-MIMO。
    • 波束赋形：分静态广播波束（覆盖）与动态业务波束（跟随用户）。波束越窄增益越高、抗干扰越好，但对信道测量与移动性跟踪要求越高。

    🎯 Massive MIMO 的配置原则

    选 32T 还是 64T、用宽波束还是窄波束，不是越大越好，而是按场景、容量、频段、终端、成本/能耗综合权衡。核心配置原则如下：

    ① 按场景选通道数（覆盖 vs 容量）

    场景推荐配置原因
    密集城区 / 高话务64T64R用户多、容量需求高，MU-MIMO 空分收益最大化。
    一般城区 / 县城32T32R容量与成本/能耗的平衡点。
    郊区 / 广覆盖8T8R / 16T16R用户稀疏，容量需求低，优先覆盖与成本。
    高楼立体覆盖64T（垂直维更多阵元）需要垂直波束赋形覆盖高层，强调 3D-MIMO。

    ② 按业务模型权衡

    • 容量受限区域（话务密集）：优先高阶 Massive MIMO，靠空分复用提升小区吞吐。
    • 覆盖受限区域（信号弱）：靠波束赋形的阵列增益拉远覆盖，通道数可适当降低。
    • 上行受限（如直播、行业上传）：关注上行通道与上行增强（如 SUL 辅助上行、上行 MU），不能只看下行。

    ③ 关键配置维度对照

    维度配置原则
    阵列结构水平阵元定方位覆盖，垂直阵元定俯仰/楼层覆盖；高楼多增加垂直维。
    波束模式覆盖优先选宽波束/少波束；容量优先选窄波束/多波束（如 SSB 波束数按场景配置）。
    MU 配对数城区可配较高的并行配对层数（如下行 8–16 层），稀疏区降低以省资源。
    频段匹配中频（3.5GHz）是 Massive MIMO 主战场；低频阵列受物理尺寸限制通道数有限。
    功率与能耗通道越多功耗越高，需结合智能关断/符号关断，按话务动态调整，控 TCO。
    终端能力空分增益依赖终端反馈（CSI）与天线数，规划时要评估在网终端的实际能力。
    一句话总结配置原则：容量看通道数与 MU 层数，覆盖看阵列增益与波束宽窄，成本看能耗与场景匹配。在话务密集城区用 64T 把容量做满，在郊区用 8T/16T 把成本压住——把钱花在最需要容量的地方，这就是 Massive MIMO 配置的精髓。

    网络切片：把网络变成"可售卖的产品"

    切片在一张物理网上切出多张端到端逻辑专网（RAN + 承载 + 核心），每张有独立 QoS/SLA 与隔离。运营商可以对外卖出"一张保障 1ms 时延的工厂专网"或"一张直播专用大带宽切片"。同一套硬件，卖出多种 SLA 等级，这正是 To B 商业模式的基础。

    MEC 边缘计算：低时延与数据本地化

    把 UPF 与算力下沉到靠近用户的边缘节点，业务数据"就近上车、就近下车"，时延从几十毫秒降到个位数毫秒，且数据不出园区——这对工业控制、机器视觉质检、AR 远程协作是刚需，也是行业客户最在意的合规点。

    CH 05进阶

    频谱与频段：5G 的"地皮"

    频谱是无线网络最稀缺的资源。频段选择直接决定覆盖范围、容量和建网成本——也是和客户谈方案时的第一性问题。

    频段类别典型频率特点适用场景
    低频 (Sub-1GHz)700MHz覆盖广、穿透强、容量小广覆盖、农村、室内补盲
    中频 (Sub-6GHz)2.6 / 3.5 / 4.9GHz覆盖与容量平衡，5G 主力城区主覆盖（黄金频段）
    高频 (mmWave)26 / 28 / 39GHz超大带宽、覆盖极小、易遮挡热点、园区、固定无线接入
    建网初期黄金法则：覆盖优先用低频/中频，先用最少的站点把面积铺开；容量热点和行业场景再叠加中高频。频段策略错了，要么钱花光覆盖还差，要么覆盖够了体验上不去。

    频率越高，为什么覆盖越差？

    电磁波频率越高，路径损耗越大、穿透与绕射能力越弱。同样的发射功率，700MHz 一个站能覆盖几公里，3.5GHz 只有几百米到一公里，毫米波甚至只有一两百米且怕遮挡。这就是为什么建网要"低频打底、中频做主、高频补热点"。

    TDD 与 FDD

    TDD（时分双工）
    上下行用同一频率、分时段切换。灵活、频谱利用高，适配下行为主的业务，5G 中频主流。典型上下行配比如 4:1、8:2。
    FDD（频分双工）
    上下行用成对的不同频率。时延稳定、上行覆盖好，多见于低频与语音。

    TDD 上下行配比可按业务调整：下行需求大就多给下行时隙；但配比影响上行覆盖与时延，需统一规划避免相邻小区交叉干扰。

    提升频谱利用的进阶手段

    • 载波聚合（CA）：把多个频段/载波"捆"在一起用，叠加带宽提升速率（如 中频 + 低频聚合）。
    • SUL 辅助上行：给中频小区借一段低频专做上行，解决"下行够、上行弱"的上行受限问题。
    • DSS 动态频谱共享：在同一频段上让 4G 与 5G 按需动态共享，平滑迁移、保护存量投资。
    • 4G/5G 协同：低频 4G 保覆盖、中频 5G 提容量，通过双连接（EN-DC）协同。
    建网频段组合建议：700MHz/低频负责广覆盖与上行兜底 + 3.5GHz承载主容量 + 毫米波/室分补热点。一句话：用低频"铺面"、中频"提速"、高频"补点"，再用 CA/SUL 把三者拧成一股绳。
    CH 06进阶

    载波聚合（Carrier Aggregation, CA）

    频谱往往是"东一块、西一块"的碎片。载波聚合就是把这些分散的频段/载波"捆成一根更粗的管道"同时为一个用户服务——带宽叠加、速率倍增，是把频谱资源价值最大化的关键技术。

    一句话理解

    载波 1
    700MHz · 覆盖好
    ＋
    载波 2
    3.5GHz · 100MHz
    ＋
    载波 3
    2.6GHz · 容量
    ＝
    聚合后
    更大带宽 / 更高速率

    每一个被聚合的载波叫成员载波（CC, Component Carrier）。理论上聚合带宽越大，峰值速率越高。

    主载波与辅载波：PCell / SCell

    PCell（主载波）
    承载控制信令、负责连接与移动性。通常选覆盖最好的频段（如低频），保证连接稳定。
    SCell（辅载波）
    按需激活/去激活，主要用来叠加数据带宽。通常选大带宽频段（如 3.5GHz）。
    激活机制
    SCell 可动态启停：有大流量时拉起、空闲时关闭以省电、降干扰。

    三种聚合方式

    类型说明典型场景
    带内连续
    Intra-band contiguous    		同一频段内相邻的连续载波聚合实现最简单，常见于中频大带宽
    带内非连续
    Intra-band non-contiguous    		同一频段内但不相邻的载波聚合利用同频段内的碎片频谱
    带间聚合
    Inter-band    		不同频段之间的载波聚合最常用：低频保覆盖 + 中频提速率

    载波聚合带来的价值

    ⚡

    峰值速率倍增

    带宽叠加，用户体验速率成倍提升，是"5G 测速好看"的直接来源。

    🧩

    盘活碎片频谱

    把零散的频段拼起来用，提升频谱投资回报，不浪费每一块"地皮"。

    📡

    覆盖与容量互补

    低频锚定覆盖、中高频提供容量，兼得稳定连接与高速率。

    ⚖️

    负载均衡

    多载波间分担用户与流量，缓解单载波拥塞，提升整网效率。

    配置原则

    • PCell 选覆盖优先频段：用低频/覆盖最稳的载波做主载波，确保连接不掉。
    • SCell 选大带宽频段：用 3.5GHz 等大带宽载波叠加吞吐，最大化速率收益。
    • 上行/下行分别规划：下行 CA 提速率；上行受限时结合 SUL 与上行 CA 改善上传体验。
    • 受终端能力约束：能聚合几个载波、哪些频段组合（band combination）取决于终端芯片能力，规划要评估在网终端。
    • 动态启停 SCell：按业务量激活/去激活辅载波，兼顾速率与节能。

    别混淆：CA 与 EN-DC / DSS

    技术本质区别
    CA 载波聚合同一制式内多载波聚合5G 聚合多个 NR 载波（或 4G 聚合多个 LTE 载波）
    EN-DC 双连接4G + 5G 跨制式双连接NSA 组网的基础，手机同时连 4G 与 5G
    DSS 频谱共享4G/5G 动态共享同一频段解决频段紧张、平滑迁移，与 CA 互补
    售前要点：CA 是"把速率做上去"最直接的牌——尤其在频谱碎片化、单载波带宽不够的市场。给客户算账时，用"低频 + 中频带间聚合"既能讲覆盖、又能讲峰值速率，是样板区出彩的关键配置。

    🗼

    AAU（有源天线单元）

    Massive MIMO 一体化天线 + 射频，如 32T32R / 64T64R。城区容量主力。

    📦

    BBU（基带单元）

    处理基带信号，可演进为 CU/DU。一个 BBU 可带多个 AAU/RRU。

    📶

    RRU（射频拉远单元）

    用于 FDD/低频或不需大规模天线的场景，配传统无源天线。

    🛰️

    无源/有源天线

    多频合一天线，减少天面占用，降低物业与风载成本。

    怎么选 AAU？（呼应第 4 章配置原则）

    AAU 选型本质就是 Massive MIMO 配置落地：密集城区选 64T64R 拼容量，一般城区 32T，郊区广覆盖 8T/16T 控成本。同时关注三项工程指标：

    • 重量与体积：直接关系到塔桅承重、抱杆改造与物业谈判，轻量化 AAU 能省大量工程费。
    • 功耗与散热：通道越多功耗越高，要配合智能关断把 OPEX 压下来。
    • 频段与带宽：匹配运营商持有的频谱（如 3.5GHz 100MHz），并预留演进空间。

    AAU 与传统 RRU 的使用场景

    同样是射频单元，AAU（有源天线，射频+天线一体）和传统 RRU（射频拉远，需外接无源天线）各有最适配的战场。选错了，要么浪费容量、要么徒增成本。

    对比维度AAU（有源天线单元）RRU + 无源天线
    天线形态射频与大规模天线一体化RRU 与无源天线分体，靠馈线连接
    核心能力Massive MIMO、波束赋形、多用户空分常规波束，无大规模空分
    容量高（空分复用）中/低
    频段多用于中频（3.5GHz 等 TDD）多用于低频 / FDD
    功耗与成本较高较低
    工程/天面集成度高、省馈线，但较重需独立天线 + 馈线，天面占用多
    🏙️

    优先选 AAU 的场景

    密集城区、高话务热点、体育场馆、CBD、高层楼宇立体覆盖——需要大容量与 3D 波束赋形的地方。中频 5G 主覆盖的首选。

    🌾

    优先选 RRU + 天线的场景

    农村与郊区广覆盖、低频段（700/900MHz）、4G 改造/共天面、用户稀疏的覆盖型站点——容量需求低、强调成本与覆盖距离。

    选型口诀：要容量、上中频 → AAU；要覆盖、上低频 → RRU + 天线。很多站点是两者并存：低频 RRU 打底覆盖，中频 AAU 叠加容量（呼应第 5 章频段组合）。

    室内覆盖（深度覆盖的关键）

    运营商 70%+ 流量发生在室内，室内覆盖是体验口碑与高 ARPU 套餐的关键，常被低估。

    🏬

    数字化室分（如 pRRU/pico）

    商场、机场、写字楼等大型建筑的室内 5G 覆盖与容量。

    🏠

    小基站 / Small Cell

    街道、热点、企业园区的轻量化补盲与容量补充。

    客户终端与行业产品

    📡

    5G CPE / FWA 终端

    把 5G 信号转成 WiFi，实现"无线家庭/企业宽带"，建网初期变现利器。

    🏭

    行业网关 / 工业模组

    面向工厂、矿山、AGV 等的工业级终端与模组。

    🧠

    MEC 边缘服务器

    园区本地算力，承载行业应用、保障数据本地化。

    🎛️

    网管 / 编排系统（OSS）

    网络规划、监控、切片编排、智能运维（含 AI 节能）。

    方案视角：真正卖给客户的从来不是单个盒子，而是"无线设备 + 承载 + 核心网 + 终端 + 平台 + 服务"的端到端方案。第 11 章给出的就是这样一份组合拳。
    CH 08精通

    网络规划与部署

    把产品变成一张能用、好用、省钱的网，靠的是规划与工程方法论。这也是和客户证明"你懂建网"的关键。

    建网三步走

    覆盖规划

    基于人口、地形、业务热力做链路预算和站址规划，确定站间距与频段组合。链路预算的核心是算清"发射功率 + 天线增益 − 路径损耗 ≥ 接收灵敏度"，由此反推单站覆盖半径与所需站数。

    容量规划

    预测用户数与忙时流量，按第 4 章原则确定 AAU 规格（32T/64T）、载波数、MU 层数与切片需求；容量不足时优先加 Massive MIMO 阶数或扩载波，而非盲目加站。

    优化与运维

    路测/MDT/MR 数据驱动参数优化（功率、倾角、邻区、波束），引入 AI 实现自动节能、容量自均衡与故障自愈。

    覆盖与容量：建网的两条主线

    问题类型典型表现解决手段
    覆盖受限边缘信号弱、掉话、上行差加低频、加站、调倾角/方位、SUL、窄波束拉远
    容量受限忙时拥塞、速率下降升 Massive MIMO 阶数、扩载波/CA、增 MU 层、做室分分流
    干扰受限SINR 差、邻区干扰频率/PCI 规划、波束协同、功率控制

    SA 还是 NSA？

    对比NSA 非独立组网SA 独立组网
    核心网复用 4G EPC全新 5G 核心网 (5GC)
    上市速度快，依托存量 4G较慢，需新建核心网
    能力主要大带宽 eMBB切片/低时延/行业全能力
    建网初期建议快速起步可选面向长期价值，强烈推荐 SA

    总拥有成本（TCO）思维

    客户真正关心的不是设备单价，而是 CAPEX + OPEX 的总账：站点租金、电费、传输、运维人力都是 OPEX 大头。一台更省电、集成度更高、运维更智能的设备，五年下来省下的钱往往超过设备差价。用 TCO 而不是单价说话，是高阶售前的标志。

    CAPEX 资本支出
    设备、天面工程、传输建设、机房改造、频谱费用。一次性投入。
    OPEX 运营支出
    电费、站址租金、传输租费、运维人力、备件。逐年发生，长期看常超过 CAPEX。
    降 TCO 的抓手
    高集成轻量化设备（省工程/塔租）、智能节能（省电）、4/5G 共站共天面（省站址）、AI 自动运维（省人力）。
    算账话术：把单价之差换算成"每月每站省多少电、少租多少天面、少派多少次维护"，再乘以站数和 5 年，客户自己就会算出该选谁。

    演进趋势：迈向 5G-Advanced

    5G 不是终点。5G-A（5.5G）正在把网络能力再推一个台阶，也是给客户描绘"投资能持续增值"的故事线。

    🚀

    万兆体验

    下行向 10Gbps、上行向 Gbps 演进，支撑 XR、裸眼 3D 等新业务。

    📍

    无源物联 (Passive IoT)

    无电池标签级连接，连接数迈向百亿，重塑仓储、零售、资产管理。

    🛜

    通感一体 (ISAC)

    网络既通信又感知，可做无人机监测、入侵检测、交通流量感知。

    🤖

    AI 内生网络

    AI 融入网络自优化、自节能、自运维，OPEX 持续下降。

    从 5G 到 5G-A 的演进逻辑

    维度5G (R15–R17)5G-A (R18+)
    下行峰值~Gbps迈向 10Gbps
    上行能力受限上行增强（大上行）
    物联NB-IoT / RedCap无源物联（Passive IoT）
    新能力切片 / 边缘通感一体、AI 内生、确定性网络
    定位米级亚米级 / 厘米级

    RedCap（轻量化 5G）是承上启下的关键：用更低成本终端覆盖摄像头、穿戴、工业传感等中速物联场景，扩大 5G 连接基数。

    给客户的关键信息：今天选择的 5G SA 架构与设备，是平滑演进到 5G-A 的底座——同一套核心网与基站通过软件升级即可获得新能力。这意味着"这笔投资不会过时"，是打消决策者"怕买早了"顾虑的有力论据。

    全球 5G-A（5.5G）：刚起步、增长快

    5G-A 是 5G 的增强版（3GPP R18 起），带来万兆体验、大上行、无源物联、通感一体等新能力。当前部署：

    • 约十余家运营商已商用 5G-A，另有 20–30 家在部署中。
    • 中国移动建设首批商用 5G-A 网络，覆盖 250+ 城市与县城，并推进规模商用；中国联通（北京）在四环内实现约 85% 的 5G-A 覆盖。
    • 中东：沙特 Zain 较早在利雅得推出 5G-A，并向多城扩展；东南亚：马来西亚、中国澳门已有商用；土耳其 Turkcell 计划 2026 年推出。
    • 前提条件：5G SA 是 5G-A 的必要基础，已上 SA 的运营商会陆续演进到 5G-A，再走向 6G。

    行业专网 / 切片培育中潜力大但目前对整体用户规模贡献有限，处于标杆项目阶段。

    CH 11实战 · GO-TO-MARKET

    实战：在 5G 建网初期的国家，如何说服客户买 5G？

    设定场景：你身处一个 5G 刚刚起步的国家。4G 还在主流、用户对 5G 没有体感、客户（运营商或政府）担心"投了没人用、收不回成本"。你的任务不是介绍技术，而是把不确定变成可算的回报。

    第一步：先认清你在卖给谁

    📶

    运营商

    关心 ARPU、市场份额、TCO、投资回收期。决策最理性。

    🏛️

    政府/监管

    关心数字经济、就业、国家竞争力、数字鸿沟。决策看战略与民生。

    🏭

    行业大客户

    港口、矿山、制造、能源。关心降本、安全、效率的具体数字。

    核心心法：建网初期客户买的不是"5G 技术"，而是"确定的商业回报"和"可控的风险"。所有话术都要落到钱、落到 ROI、落到可分期的投入。

    第二步：核心要考虑的因素（说服的支柱）

    维度客户的真实顾虑你的应对策略

    第三步：分阶段的说服路径（从赚钱到生态）

    用 FWA 证明"5G 当年就能赚钱"

    很多建网初期国家固网宽带渗透率低。5G FWA（CPE）无需挖沟铺光纤，几天即可开通家庭/中小企业宽带——这是最快变现、说服力最强的第一仗。

    建样板区，制造"看得见的体验"

    在首都核心区、机场、大学做高质量覆盖样板，用测速、直播、云游戏制造口碑和媒体效应，拉动消费者认知与高 ARPU 套餐。

    切入一个标杆行业专网

    选一个本国支柱产业（如港口、矿山、油气），做一个降本增效的标杆项目，用真实数字（效率+30%、人力-X 人）撬动整个行业。

    绑定国家数字战略，做大生态

    把 5G 定位为数字经济底座，联合政府、终端、内容、行业伙伴，争取频谱、补贴与政策，形成正循环。

    用 TCO 与演进路标锁定长期合作

    以五年 TCO 和向 5G-A 的演进承诺，把单次设备采购变成长期战略伙伴关系。

    第四步：要售卖的产品组合（端到端方案）

    ① 无线网络设备

    网络底座 · CAPEX 主体
    • 低/中频宏站 AAU（如 64T64R）：城区容量与覆盖主力。
    • 低频 RRU + 多频天线：广覆盖、农村与室内补盲，用最少站点铺面。
    • BBU / CU-DU：基带核心，支持 4/5G 共站、平滑演进。
    • 数字化室分 + 小基站：商场、机场、园区的深度覆盖。

    ② 终端 —— 最快变现的入口

    现金流 · 见效最快
    • 5G CPE / FWA 终端：无线家庭/企业宽带，建网初期的"印钞机"。
    • 5G 行业网关与工业模组：面向工厂、AGV、摄像头的连接终端。
    • 补贴版消费终端生态：联合手机厂商降低用户换机门槛。

    ③ 核心网与承载

    能力使能 · 切片基础
    • 5G 核心网（5GC，SA）：解锁切片、低时延、行业能力的前提。
    • 承载/传输（IP+光）：连接 RAN 与核心网的高速管道。
    • MEC 边缘计算：行业数据本地处理、低时延、数据不出园区。

    ④ 平台、软件与服务 —— 高毛利、强粘性

    长期收入 · 锁定客户
    • 网络切片与编排平台：把网络变成可按 SLA 售卖的产品。
    • 智能网管 / AI 节能 / 自动运维（OSS）：直接降 OPEX，是 TCO 故事的关键。
    • 行业应用方案：智慧港口、智能矿山、智能制造等打包解决方案。
    • 咨询、融资、运维外包：商业模式设计 + 融资方案 + 共担风险，降低客户决策门槛。
    整章总结：在建网初期，不要卖技术参数，要卖"分阶段、低风险、能算账"的赚钱路径。顺序是：用 FWA 终端 快速变现 → 用 样板区 建口碑 → 用 行业专网（切片+MEC） 做高价值标杆 → 用 平台与服务 锁定长期。产品则按"无线设备 + 终端 + 核心网/承载 + 平台服务"端到端打包，用 TCO 与演进路标证明这是一笔会增值、不会过时的投资。

    RAN 架构路线抉择：D-RAN / C-RAN / Cloud RAN / Open RAN / AI-RAN

    对 CTO 而言，"怎么建"先于"建什么"。RAN 架构决定了未来 5–8 年的 TCO、供应商格局、运维能力与演进自由度。这一章把几条主流路线、各自的收益与代价、以及决策方法摆清楚。

    架构形态核心收益代价 / 风险

    C-RAN 集中式BBU 集中池化小区协同、省机房电力、易演进需大带宽低时延前传

    Open RAN (O-RAN)CU/DU/RU 接口开放、白盒多厂商解耦、避免锁定、创新加速集成复杂、责任界面模糊、成熟度参差

    Open RAN：收益与代价要一起看

    ✅

    收益

    打破单一厂商锁定、引入多厂商竞争降本、白盒硬件、接口开放带来创新与议价权，符合部分国家的供应链自主诉求。

    ⚠️

    代价

    多厂商集成与互操作复杂、故障定位与责任界面模糊、当前在大规模容量/功耗上仍弱于一体化方案、需要更强的自有集成与运维能力。

    Cloud RAN / vRAN 与 AI-RAN 的方向

    vRAN 把 CU/DU 软件化跑在通用服务器（配 L1 加速卡），获得云的弹性与解耦；AI-RAN 更进一步，让基站算力同时承载 RAN 与 AI 推理业务——既用 AI 反哺网络（节能、优化），又把边缘算力作为新营收（边缘 AI 服务）。这是面向 6G 的重要趋势，但目前仍处早期。

    决策维度与决策树

    • 前传/时延条件：有充足光纤资源才适合 C-RAN/集中化，否则 D-RAN 更现实。
    • 自有集成与运维能力：能力强、追求自主 → 可试点 O-RAN/vRAN；能力有限 → 一体化方案更稳。
    • 供应链与地缘策略：需多厂商/避免锁定 → O-RAN 价值大。
    • 规模与节奏：建网初期求快求稳 → 成熟一体化；中长期求开放与弹性 → 逐步引入云化/开放。
    • 演进诉求：看重 AI 与 6G → 关注 AI-RAN/vRAN 的算力底座。
    建议框架：建网初期、能力有限 → 选成熟一体化（D-RAN/C-RAN）快速、稳定起网；中长期、追求开放与避免锁定 → O-RAN/vRAN 试点先行，但务必先定清晰的集成主体与责任界面——Open RAN 失败的项目，大多败在"谁为端到端性能负责"没说清。

    安全与数据主权

    安全是 CTO 的第一道闸。5G 是承载工厂、电网、医疗的生产网，一旦被攻破，后果远超消费网。这一章覆盖 5G 内生安全、切片隔离、数据主权与厂商可信。

    5G 内生安全：比 4G 强在哪

    用户身份加密
    永久标识 SUPI 经公钥加密为 SUCI 才上空口，根治 4G 的 IMSI 捕获/伪基站。
    双向认证
    5G-AKA / EAP-AKA′，网络与终端互相认证，防伪基站与伪终端。
    空口安全
    信令面与用户面均支持加密 + 完整性保护。
    服务化接口安全
    核心网 NF 之间 TLS 加密 + OAuth2.0 授权，防内部横向渗透。
    漫游安全
    SEPP 在运营商边界做信令过滤与保护，防跨网信令攻击。

    网络切片的安全隔离

    切片不仅是性能隔离，也是安全隔离。按需求分级：逻辑隔离（共享资源、软隔离）→ 资源预留（独享 PRB/算力）→ 物理专用（专用频率/UPF/硬件）。行业专网应做到 RAN + 传输 + 核心的端到端隔离，防止切片间串扰与攻击扩散。

    数据主权与本地化

    通过 UPF / MEC 下沉，可实现"数据不出园区、不出国境"——业务数据在本地处理与存储，满足 GDPR、数据驻留法规与行业合规要求。这是政府与大型行业客户最在意的合规底线，也是赢得监管信任的关键卖点。

    新增攻击面与对策

    新风险来源对策
    海量 IoT 僵尸网络 / DDoSmMTC 海量弱终端被劫持终端认证、限速、流量异常检测、隔离
    切片间攻击共享资源隔离不足强隔离 + 零信任 + 切片安全策略
    边缘节点物理暴露MEC 下沉到园区/路边边缘加固、可信计算、本地加密
    供应链 / 固件后门设备与软件供应链源码审计、可信供应链、SBOM、安全认证
    信令风暴大量终端并发接入核心网过载保护与拥塞控制

    厂商可信与地缘

    设备选型不只看性能，还要看可信：是否通过第三方安全认证（如 GSMA NESAS / 3GPP SCAS）、是否可审计、责任是否清晰、是否符合本国监管要求。多厂商策略可降低单一依赖，但会提高集成与安全管理复杂度——这是 CTO 要权衡的战略选择。

    CTO 安全清单：① 切片隔离做到哪一级？② 关键数据是否本地化、可证明？③ 设备是否有第三方安全认证？④ 边缘节点如何加固？⑤ 是否有端到端的应急响应与审计能力？把这五问写进招标与验收，比任何承诺都有效。

    设备单价只是冰山一角。按 5 年看，OPEX（尤其电费）常常超过 CAPEX。这一章给 CTO 一套能算账的模型，把"更省电"翻译成"省多少钱"。

    5 年 TCO 模型

    用一个简单公式跟客户对齐口径：

    TCO = CAPEX（设备 + 工程安装 + 频谱费 + 传输建设） + Σ 5年 OPEX（电费 + 站址租金 + 传输租费 + 运维人力 + 备件）。
    投资回收期 = 累计净现金流由负转正所需时间。

    降 TCO 的四个抓手：高集成轻量化设备（省工程与塔租）、智能节能（省电）、4/5G 共站共天面（省站址）、AI 自动运维（省人力）。一台更省电、更集成的设备，五年省下的常超过设备差价。

    $/bit：长期盈利的生命线

    流量指数增长但 ARPU 基本走平，唯一的出路是单位比特成本（$/GB）持续下降——靠更高频谱效率（Massive MIMO/CA）、更省电的设备、更高的自动化。CTO 评估任何方案，最终都应落到"它能不能把 $/bit 打下来"。

    算账话术：把单价之差换算成"每月每站省多少电、少租多少天面、少派多少次维护"，再 × 站数 × 5 年。让客户自己用 TCO 模型得出结论，比你报任何参数都有说服力。

    资深 CTO 信的不是"没问题"，而是"你知道哪里会出问题、怎么兜底"。这一章把 5G 项目最常见的坑、根因与对策列清楚——这往往比任何技术介绍都更能赢得信任。

    场景化部署策略：场馆 · 农村 · 景区 · 机场

    没有"万能配置"，只有"对的场景配对的网"。同一套工具箱（频段、设备、MIMO、室分、切片）在不同场景下要配出完全不同的方案。这一章给一套可复用的判断方法，再逐个拆解典型场景。

    通用方法论：四个问题推导任何场景

    面对一个陌生场景，先问清四件事，配置自然浮现：

    👥

    ① 用户密度与峰值

    多少人、并发多大？决定 MIMO 阶数、载波数与是否需要专用容量层。

    🔀

    ② 业务特征

    上行还是下行为主？时延/可靠性要求？决定是否上行增强、是否切片/URLLC。

    🗺️

    ③ 覆盖环境

    室内/室外、地形、穿透？决定频段组合与设备形态（AAU/RRU/室分/小站）。

    ⏱️

    ④ 流量时间分布

    是否潮汐/事件型？决定是否需要临时扩容、智能节能与应急保障。

    推导链：密度→MIMO 阶数与容量层；业务→上行增强/切片；环境→频段与设备形态；时间→扩容与节能策略。把这四问套到任何场景，都能快速给出方案骨架。

    🏟️ 场馆 / 体育场

    场景特征：万人级超高密度、事件型潮汐（平时空、赛演时爆满）、海量上行（观众集中直播/上传）、看台金属结构强反射。

    部署策略：

    • 高阶 Massive MIMO 64T + 窄波束分区覆盖看台，按区块做空分容量。
    • 大上行增强（超级上行 / 上行 CA）——这是场馆的命门，观众都在往外传。
    • 专用容量层（独立大带宽载波）+ 数字化室分覆盖大厅/通道。
    • 事件期临时扩容（应急通信车/临时载波），平时智能关断省电。
    • MEC 支撑 AR 观赛、即时回放、多视角直播；为转播配媒体专用切片。
    推荐配置：中频 3.5GHz 主容量 + 64T 分区波束 + 数字化室分 + 上行增强 + MEC/切片。

    🌾 农村 / 广域覆盖

    场景特征：用户稀疏、ARPU 低、成本极敏感，电力与传输受限，覆盖面积大。

    部署策略：

    • 覆盖优先：用 700MHz 低频大站距、超远覆盖，用最少站点铺面。
    • 设备降本：RRU + 无源天线（不上昂贵 AAU）、低功耗、共站共塔。
    • FWA 优先变现：用 5G CPE 替代缺失的固网宽带，是农村最快的现金流。
    • 工程适配：太阳能/混合供电、微波或卫星回传、轻量化简易站。
    推荐配置：低频 FDD + 8T/16T RRU + 多频天线 + FWA CPE；目标是"广覆盖 + 低 TCO + 快变现"。

    ⛰️ 景区 / 旅游区

    场景特征：地形复杂（山、水、林）、节假日潮汐暴增、景观与环保限制（不能立大铁塔）、热点分散（观景台、索道、入口）。

    部署策略：

    • 频段：低频保大面覆盖 + 中频补热点（观景台、广场、入口）。
    • 美化/隐形站点：仿生树、景观杆、塔桅美化，兼顾覆盖与景观。
    • 潮汐扩容：节假日应急车/临时载波，淡季智能关断。
    • 大上行支撑游客直播打卡；叠加智慧景区（视频监控、客流统计、无人机巡检、AR 导览，配切片/MEC）。
    推荐配置：低频广覆盖 + 中频热点 + 美化天线 + 上行增强 + 智慧景区切片。

    ✈️ 机场

    场景特征：超大室内航站楼、高密度高价值用户、运营关键业务（廊桥/行李/安检/AGV/地勤）、需严格干扰控制（邻近航管雷达等敏感系统）、公网 + 专网并存。

    部署策略：

    • 室内以数字化室分为主，覆盖出发/到达大厅、廊桥、值机区。
    • 公网容量 + 行业专网（切片/独享 UPF）：承载行李追踪、视频安防、AGV、地勤调度。
    • 干扰协调：规避/协调航管频段，3.5GHz 与雷达邻频要严控指标。
    • MEC 实现运营数据本地化与低时延；停机坪用宏站定向覆盖。
    推荐配置：数字化室分为主 + 专网切片/MEC + 严格干扰指标 + 室外定向宏站。

    🚄 交通沿线（高铁 / 地铁 / 高速）

    场景特征：高速移动（频繁切换、多普勒）、车厢穿透损耗大、隧道封闭、线状覆盖。

    部署策略：沿线专网建设、超级小区/小区合并减少切换、隧道用漏缆或分布系统、移动性参数优化、必要时车载中继（车地系统）。

    🏭 工业园区 / 工厂

    场景特征：确定性控制、数据需本地化、设备密集、室内外混合。

    部署策略：5G 专网（切片或独享频率 + 独立 UPF/MEC）、URLLC 低时延高可靠、室分 + 小基站做深度覆盖、数据不出园区保障合规。

    网络仿真与覆盖规划：如何更好地规划 5G 覆盖

    建网前先在计算机里"建一遍"——这就是网络仿真。它用电子地图 + 传播模型预测覆盖、容量与质量，让你在浇筑混凝土之前就把站址、天线和参数调到最优。仿真做得准，CAPEX 省得多、返工少。

    仿真在规划中的位置：一个闭环

    数据准备
    3D 地图/业务热力/站址库
    ›
    模型校正
    传播模型 + 实测校准
    ›
    站址仿真
    布站/天线/参数
    ›
    覆盖预测
    RSRP/SINR/速率
    ›
    实测反馈
    路测/MDT 校正

    关键是闭环：仿真 → 建设 → 实测 → 回头校正模型与参数。仿真不是一次性动作，而是持续逼近真实的过程。

    三类传播模型怎么选

    模型类型代表精度计算量适用
    经验模型Okumura-Hata / COST-231低–中小快速估算、农村广域
    半确定性SPM 标准传播模型中–高（需校正）中主流规划（城区/郊区）
    确定性 / 射线追踪Ray Tracing高大密集城区、室内、毫米波、3D

    频段越高（3.5GHz/毫米波）、环境越复杂（密集楼宇/室内），越需要射线追踪与 3D 建模；农村广覆盖用经验模型即可快速估算。

    链路预算：仿真背后的"算术"

    覆盖半径的本质是一道不等式：发射端能量 − 一路损耗 ≥ 接收端灵敏度。算出最大允许路损（MAPL），再反推单站覆盖半径。

    发射侧（+）
    基站 EIRP = 发射功率 + 天线增益 + 波束赋形/Massive MIMO 增益
    损耗侧（−）
    路径损耗 + 穿透损耗 + 人体损耗 + 阴影衰落余量 + 干扰余量
    接收侧（门限）
    接收灵敏度（与噪声、解调门限相关）
    5G 关键点
    上行接收波束赋形增益对上行覆盖至关重要；覆盖往往由上行决定

    仿真输出的关键指标

    指标含义衡量什么
    SS-RSRP同步信号参考信号接收功率覆盖强度（信号有多强）
    SS-SINR信号与干扰加噪声比信号质量（能跑多快）
    吞吐率下行/上行速率分布图用户体验
    覆盖率满足门限的面积/人口比例整体覆盖达标度
    边缘速率小区边缘用户速率体验底线（最该被保障）

    5G 仿真的特殊难点

    🎯

    Massive MIMO / 波束建模

    动态波束与多用户空分难以精确仿真，增益依赖信道与终端能力——要建模但别高估。

    📡

    高频段挑战

    3.5GHz/毫米波路损与穿透损耗大、绕射弱，需 3D 地图与射线追踪才准。

    ⬆️

    上行受限

    上下行覆盖不平衡，下行够、上行弱很常见——应以上行链路定小区边缘。

    🏙️

    3D 立体覆盖

    高楼需垂直维波束与逐层仿真，平面预测会漏掉中高楼层。

    模型校正：仿真准不准的命门

    仿真是"垃圾进、垃圾出"。两件事决定精度：① 电子地图分辨率（地形 DEM + 建筑 DSM + 地物 clutter）；② 传播模型校正——用 CW 单载波测试或路测数据对模型参数做 tuning，让预测贴近实测。建网后再用路测/MDT 数据持续反哺，模型越用越准。

    常用仿真工具

    室外宏网常用 Atoll（Forsk）、Planet（Infovista）、ASSET（TEOCO）、Ranplan；射线追踪用 Wireless InSite；室内分布用 iBwave；此外各设备厂商均有自有规划平台。工具是手段，数据与模型校正才是精度的来源。

    如何更好地规划 5G 覆盖（实操八条）： ① 用高精度 3D 电子地图（地形+建筑+地物）； ② 选对模型并用 CW/路测校正，别用默认参数； ③ 链路预算先行，以上行定小区边缘； ④ 频段分层：低频保覆盖、中频提容量； ⑤ 不只仿真覆盖，还要仿真容量与 SINR； ⑥ 合理建模 Massive MIMO 波束增益，不高估； ⑦ 室内场景单独仿真（iBwave + 建筑 3D）； ⑧ 坚持仿真—建设—实测—校正闭环。
