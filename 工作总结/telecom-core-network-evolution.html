OVERVIEW

导言：什么是核心网，它为什么不断演进

Mobile Core Network · The Brain of the Telecom System

如果把移动通信网络比作人体，无线接入网（RAN）是触达世界的"四肢与感官"，承载网是"血管"，那么核心网（Core Network, CN）就是统筹调度的"大脑与中枢神经"。它决定了你是谁、能不能上网、连到哪、按什么策略和资费、以及通话与数据如何被路由。

核心网的六大基本职责

无论技术如何代际更迭，核心网始终围绕以下功能展开。理解这六点，就能看懂任何一代架构图里每个网元"在忙什么"。

会话管理

认证与鉴权

一条贯穿四十年的演进主线

Key Idea 核心网演进的本质，是"不断解耦"：把原本捆在一台专用设备里的功能，一层层拆开、再灵活重组。 电路与分组解耦 → 控制与承载解耦 → 软件与硬件解耦 → 功能与功能之间服务化解耦。每一次解耦，都换来更高的弹性、更低的成本和更快的业务创新。

TIMELINE

演进主线与关键里程碑

The Evolution Roadmap · 1990s → 2030s

核心网的每一次跃迁，都由"业务需求"和"使能技术"双轮驱动：从打电话，到上网，到万物互联，到智能内生。下面这条时间轴是理解全书的骨架。

1991 · 2G
GSM 数字电路交换核心网
以 MSC 为中心的电路域（CS），数字化语音与短信。核心网=电话交换网的移动版。
1997-2000 · 2.5G
GPRS 引入分组域（PS）
新增 SGSN/GGSN，首次把"分组数据/上网"带入移动核心网。CS 与 PS 双域并存格局确立。
2001-2005 · 3G
UMTS 与 R4 软交换
R4 将 MSC 拆为 MSC Server（控制）+ MGW（承载）——控制与承载分离的首次工程实践。R5 引入 IMS。
2009-2012 · 4G
EPC：全IP、扁平化、取消CS域
MME/SGW/PGW/HSS/PCRF 登场，语音改由 VoLTE（IMS）承载。R14 引入 CUPS，为5G铺路。
2018-2020 · 5G
5GC：服务化架构 SBA
网络功能（NF）以微服务+HTTP/2 互调，UPF 用户面下沉边缘，网络切片、能力开放成为标配。
2030+ · 6G
AI 原生 · 通感算一体 · 空天地一体
网络内生智能与安全、确定性时延、算力与连接融合、卫星与地面网络统一架构（探索中）。

两条永恒的技术暗线

① 业务驱动：连接对象的变化
  • 2G：人与人通话（Voice First）
  • 3G/4G：人与内容（Mobile Internet）
  • 5G：人、物、行业（eMBB/URLLC/mMTC）
  • 6G：人、物、智能体与数字孪生世界
② 技术驱动：架构形态的变化
  • 专用电路交换机（封闭一体机）
  • 分组化 + 软交换（控制承载分离）
  • 全IP + NFV 虚拟化（软硬解耦）
  • 服务化 + 云原生（微服务、容器）

2G · 1991

2G GSM/GPRS 核心网

Circuit-Switched Core + Packet-Switched Domain (GPRS)

2G 核心网本质上是一张"会移动的数字电话网"。它的设计目标是高可靠地接通语音呼叫，因此采用电路交换（Circuit Switched, CS）——为每通电话独占一条端到端的时隙通道。后期的 GPRS 才补上了分组数据能力。

MSC
Mobile Switching Center
核心交换节点，负责呼叫建立、路由、切换、计费。它就是移动版的电话交换机。
GMSC
Gateway MSC
网关 MSC，负责与 PSTN 等外部网络互通，呼入时先查 HLR 再选路。
HLR
Home Location Register
归属位置寄存器，永久存储用户签约数据与当前所在 VLR 地址。是用户数据的"户籍中心"。
VLR
Visitor Location Register
访问位置寄存器，缓存当前服务区内用户的临时数据，通常与 MSC 合设。
AuC
Authentication Center
鉴权中心，存储 Ki 密钥并生成鉴权三元组，是安全的根。常与 HLR 合设。
EIR
Equipment Identity Register
设备识别寄存器，按 IMEI 维护黑/白/灰名单，拦截被盗或非法终端。

分组域（PS Domain）—— GPRS / 2.5G

为了让手机能"上网"，GSM 在 CS 之外叠加了 GPRS 分组域，引入两个关键网元。这是移动核心网第一次拥有"互联网基因"。

SGSN
Serving GPRS Support Node
服务 GPRS 支持节点：分组域的"MSC"，负责移动性管理、会话管理、对分组数据计费与鉴权。
GGSN
Gateway GPRS Support Node
网关 GPRS 支持节点：分组域出口，分配 IP 地址、连接外部 PDN（互联网），是 4G PGW 的前身。
信令底座2G/3G 核心网信令运行在 SS7（七号信令） 之上，移动相关流程使用 MAP（Mobile Application Part）协议。这是一套与互联网 TCP/IP 完全不同的电信专用信令体系。
本代要点CS 与 PS 双域并存是 2G/3G 的核心特征：语音走电路域、数据走分组域，两套独立网元、两套独立流程。后续演进的一大主题，正是逐步淘汰 CS 域、走向全分组（All-IP）。
3G · 2001

3G UMTS 核心网与软交换变革

UMTS Release 99 → R4 Softswitch → R5 IMS

3G（以 WCDMA/UMTS 为代表）在核心网上最初是 2G 的"平滑升级"，但很快通过 R4 版本引入了影响深远的架构革命——软交换，第一次把"控制"和"承载"在工程上彻底分开。

R99：继承式架构

3GPP Release 99 基本沿用 2G 的核心网结构，CS 域仍是 MSC/VLR/GMSC + HLR，PS 域仍是 SGSN/GGSN，只是升级以支持更高速率。变化主要在接入网：用 UTRAN（RNC + NodeB）取代了 GSM 的 BSS。

R4：软交换——控制与承载分离 ★

这是核心网演进史上的关键转折。传统 MSC 一台设备同时干两件事：呼叫控制信令（决定怎么接通）和媒体承载交换（搬运语音流）。R4 把它一拆为二：

传统 MSC（一体机）
  • 控制 + 承载捆绑在同一硬件
  • 承载基于 TDM 电路，扩容昂贵
  • 新增业务需整机升级
  • 控制能力与端口数强耦合
R4 软交换（控制/承载分离）
  • MSC Server：只管呼叫控制信令
  • MGW（媒体网关）：只管语音媒体流交换
  • 承载转向 IP/ATM 分组传输
  • 控制与承载可独立扩容、集中部署

为什么重要"控制与承载分离"的思想从这里发芽，一路延续到 4G 的 CUPS 和 5G 的 UPF 独立用户面。可以说，R4 软交换是 5G 架构的祖先之一。

R5/R6：IMS 登场

Release 5 引入 IMS（IP Multimedia Subsystem，IP 多媒体子系统），用一套基于 SIP 协议的 IP 化框架来统一承载语音、视频、消息等多媒体业务。IMS 当时未被广泛商用，却成为 4G 时代 VoLTE 的技术基础。

CSCF
Call Session Control Function
呼叫会话控制功能，分 P/I/S-CSCF，是 IMS 的 SIP 信令大脑。
HSS
Home Subscriber Server
归属用户服务器，IMS 的用户数据库，也是 HLR 向 4G HSS 演进的雏形。
MGCF
Media Gateway Control Function
媒体网关控制功能，负责 IMS 与传统 PSTN/CS 网络的互通。
4G · 2009

4G EPC：全IP、扁平化、取消CS域

Evolved Packet Core (EPC) · System Architecture Evolution (SAE)

4G 核心网 EPC 是一次彻底的"瘦身与提速"。它做了一个大胆决定：取消电路交换域，全网 IP 化。语音不再有专用电路，而是作为一种特殊的数据业务（VoLTE）跑在分组网上。同时网络层级被压扁，时延大幅下降。

五大核心网元

MME
Mobility Management Entity
控制面核心。处理 NAS 信令、用户鉴权、移动性管理、寻呼、承载管理与 SGW/PGW 选择。相当于 4G 的"指挥中心"。
S-GW
Serving Gateway
用户面本地锚点。转发用户数据包，处理 eNodeB 间切换时的数据路由，是用户面的"中转站"。
P-GW
PDN Gateway
用户面出口锚点。分配 IP 地址、连接外部数据网、执行策略与计费（PCEF）、深度包检测。GGSN 的演进体。
HSS
Home Subscriber Server
归属用户服务器。融合了 HLR+AuC，集中存储签约数据与鉴权信息。通过 S6a/Diameter 与 MME 交互。
PCRF
Policy & Charging Rules Function
策略与计费规则功能。动态下发 QoS、限速、计费策略到 P-GW 中的 PCEF 执行，实现精细化运营。

语音怎么办？—— VoLTE 与回落

既然 EPC 没了 CS 域，语音就需要新方案：

  • VoLTE（Voice over LTE）：语音作为 IMS 上的 VoIP 业务，跑在 LTE 分组网上，高清音质，是目标方案。
  • CSFB（CS Fallback）：来电话时把终端临时"回落"到 2G/3G 的 CS 域接听，过渡方案。
  • SRVCC（Single Radio Voice Call Continuity）：VoLTE 通话移动到无 LTE 覆盖区时，无缝切换到 2G/3G CS，保证不断线。

CUPS：为 5G 埋下伏笔（R14）

承上启下CUPS（Control and User Plane Separation，控制面与用户面分离）在 R14 引入，把 S-GW、P-GW 各自拆成 控制面（SGW-C/PGW-C） 与 用户面（SGW-U/PGW-U）。用户面可以独立下沉、靠近用户部署。这正是 5G UPF 灵活部署、边缘计算的直接前身。
CUPS 之前
  • 控制与转发功能耦合在网关内
  • 用户面位置固定，难以下沉
  • 容量按整机扩展
CUPS 之后
  • 控制面集中、用户面分布式
  • 用户面可部署到网络边缘，降时延
  • 控制与转发独立弹性扩容

    5G · 2018

    5G 核心网 5GC：服务化架构 SBA

    5G Core · Service-Based Architecture

    5GC 是核心网历史上最彻底的一次重构。它不再画"一个个盒子用线连起来"的传统架构图，而是把每个网络功能（NF）变成一个可被发现、可被调用的微服务，挂在一条公共总线上互相通信。这就是 SBA（Service-Based Architecture，服务化架构）。

    从"点对点接口"到"服务总线"

    4G 及以前：刚性点对点
    • 网元间用固定接口（S11/S6a…）两两相连
    • 新增一个网元，要改一堆接口
    • 协议各异（Diameter/GTP/SS7）
    5G SBA：服务化总线
    • NF 把能力封装成"服务"注册到总线
    • 调用方通过 NRF 发现并按需调用
    • 统一用 HTTP/2 + REST + JSON
    • 即插即用、独立升级、云原生友好

    核心网络功能（NF）一览

    AMF
    Access & Mobility Mgmt Function
    接入与移动性管理。承接 4G MME 的移动性部分：注册、连接、移动性、寻呼、NAS 信令。
    SMF
    Session Management Function
    会话管理。负责 PDU 会话建立、IP 地址分配、并通过 N4 控制 UPF。承接 MME/SGW-C/PGW-C 的会话部分。
    UPF
    User Plane Function
    唯一用户面功能。数据包路由转发、QoS 执行、计费统计。可灵活下沉到边缘，是低时延与 MEC 的关键。
    AUSF
    Authentication Server Function
    鉴权服务功能，负责 5G 接入鉴权流程，与 UDM 配合完成安全认证。
    UDM
    Unified Data Management
    统一数据管理，5G 的"HSS"，处理签约、鉴权数据、用户标识。计算逻辑与存储分离。
    UDR
    Unified Data Repository
    统一数据仓库，真正存放签约/策略/结构化数据，供 UDM/PCF 等读写。实现"计算-存储分离"。
    PCF
    Policy Control Function
    策略控制功能，4G PCRF 的演进，统一管理 QoS、计费、接入与移动性策略。
    NRF
    Network Repository Function
    网络存储功能，SBA 的"服务注册与发现中心"。所有 NF 在此注册，调用方在此查找。是 5GC 独有的灵魂网元。
    NSSF
    Network Slice Selection Function
    网络切片选择功能，根据业务为终端选择合适的网络切片实例。
    NEF
    Network Exposure Function
    网络能力开放功能，把核心网能力安全地开放给第三方应用/AF，是 5G to B 的桥梁。

    5GC 的三大杀手锏

    网络切片
    Network Slicing
    在同一物理网络上切出多张相互隔离的逻辑专网，分别面向 eMBBURLLCmMTC 等不同场景，各有独立 QoS 与资源。
    边缘计算 MEC
    Multi-access Edge Computing
    借助 UPF 下沉，把数据处理放到靠近用户的边缘节点，实现毫秒级时延，支撑工业控制、自动驾驶、AR/VR。
    能力开放
    Capability Exposure
    通过 NEF 将网络能力 API 化对外开放，让行业应用按需调用 QoS、定位、切片等，催生 5G to B 商业模式。

    SA 与 NSA：两种组网选择

    NSA 非独立组网
    • 5G 基站锚定在 4G EPC 核心网上
    • 快速部署，复用 4G 投资
    • 仅增强带宽，难用切片/URLLC
    SA 独立组网
    • 5G 基站直连全新 5GC 核心网
    • 完整释放切片、MEC、URLLC 能力
    • 是 5G 价值的真正载体
    易混淆点很多人以为"上了 5G 就是 5GC"。其实 NSA 阶段的 5G 仍在使用 4G 的 EPC 核心网，只有切换到 SA 独立组网 才真正用上 5GC 服务化架构与全部新能力。

      PRINCIPLES

      贯穿四代的三大架构理念

      Three Great Decouplings That Shaped the Core

      如果只记住一件事，那就是：核心网演进史，是一部"解耦史"。把这三条主线看透，任何一代架构图都能秒懂。

      ① 业务解耦：电路域 → 分组域（CS → PS）

      从 2G 的电路交换，到 GPRS 引入分组域双域并存，再到 4G EPC 干脆取消 CS 域、全网 IP 化。语音从"独占电路"变成"分组网上的一种数据业务（VoLTE）"。这是从"电话网思维"到"互联网思维"的根本转变。

      ② 转发解耦：控制与承载分离（CUPS）

      这条线索贯穿最久：3G R4 软交换（MSC Server / MGW）→ 4G CUPS（SGW/PGW 各拆控制面与用户面）→ 5G 把用户面彻底独立为 UPF。好处是用户面能灵活下沉到边缘、控制面能集中云化。

      "控制/承载分离"的接力演进
      R4 软交换MSC-S / MGW
      4G CUPSxGW-C / xGW-U
      5G SBASMF / UPF

      ③ 软硬解耦 + 功能解耦：虚拟化与服务化

      两个并行的趋势在 5G 汇合：

      • 软硬解耦（NFV）：网络功能从专用电信设备里"软件化"出来，跑在通用 x86 服务器/云上。一台设备不再只能干一件事。
      • 功能解耦（SBA）：网元之间从刚性点对点接口，变成总线上互相调用的服务，可独立扩缩容、升级、即插即用。

      一句话总结从"专用、刚性、集中、电路"走向"通用、弹性、分布、分组"——四个词，概括四十年。
      CLOUD

      核心网云化：NFV、SDN 与云原生

      Telco Cloud · From Virtualization to Cloud-Native

      5GC 的服务化只有跑在"云"上才能发挥威力。支撑这一切的，是三套关键技术：NFV（虚拟化）、SDN（网络可编程）、以及云原生（容器与微服务）。它们共同把核心网从"一柜柜专用机"变成"一片可弹性调度的软件"。

      NFV：网络功能虚拟化

      NFV（Network Functions Virtualization）的核心思想：把过去固化在专用硬件里的网络功能，变成可在通用服务器上运行的软件（VNF）。由 ETSI 定义的标准框架包含三层：

      VNF
      Virtualized Network Function
      虚拟化网络功能，即软件化的网元（如虚拟 AMF/SMF）。
      NFVI
      NFV Infrastructure
      承载 VNF 的基础设施：通用计算、存储、网络 + 虚拟化层。
      MANO
      Management & Orchestration
      管理与编排：负责 VNF 的生命周期、资源调度与自动化部署。

      SDN：软件定义网络

      SDN（Software Defined Networking）把网络设备的控制面与转发面分离，用集中的控制器统一编程下发转发规则。它与核心网"控制/承载分离"的理念同源，让网络连接像软件一样可编程、可灵活调度——尤其在数据中心内部和切片承载中作用突出。

      从虚拟机到云原生（Cloud Native）

      NFV 早期用虚拟机（VM）承载 VNF，较重。5G 时代进一步走向容器化：网络功能被拆成微服务、打包成容器、由 Kubernetes 编排，称为 CNF（Cloud-Native Network Function）。

      承载形态演进
      专用硬件物理网元
      VM / VNFNFV 虚拟机
      容器 / CNF云原生微服务
      VNF（虚拟机时代）
      • 整网元打包成一个虚拟机镜像
      • 启动慢、资源占用大
      • 弹性伸缩粒度粗
      CNF（云原生时代）
      • 拆成微服务、容器化部署
      • 秒级弹性、按需扩缩单个服务
      • 配合 CI/CD 实现持续交付
      • 故障自愈、灰度升级
      Telco Cloud运营商为此构建专门的电信云：在中心 DC 部署控制面 NF，在边缘 DC 部署 UPF 等用户面，配合 MANO 与 K8s 实现全网自动化编排。这就是"核心网上云"的工程落地形态。
      价值闭环软硬解耦（NFV）让功能上云 → 服务化（SBA）让功能可灵活组合 → 云原生（CNF）让功能弹性敏捷。三者叠加，核心网才真正具备了"软件定义、按需生长"的能力。
      6G · 2030+

      6G 展望：面向智能内生的核心网

      Toward 6G · AI-Native & Converged Core (Exploratory)

      6G 仍处于研究与标准前期，架构尚未冻结。但业界方向已逐渐清晰：在 5G 服务化与云原生的基础上，让网络内生智能、内生安全，并把通信、感知、计算、卫星深度融合。以下为当前主流探索方向，供前瞻参考。

      阅读提醒本章内容为面向未来的技术趋势与研究方向，并非已冻结的标准。具体网元名称与架构以未来 3GPP 正式版本为准。

      六个主要探索方向

      AI 原生
      AI-Native Network
      AI 不再是外挂工具，而是内嵌于核心网架构本身：网络自优化、自配置、自愈，意图驱动运维。可能出现专门的网络智能功能（如 NWDAF 的演进）。
      确定性网络
      Deterministic Networking
      提供有界、可保证的端到端时延与抖动，支撑工业全闭环控制、远程手术、全息通信等严苛场景。
      通感算一体
      ISAC + Computing
      通信与感知融合（用信号"看"环境），并将算力作为网络原生资源，连接与计算统一编排（算力网络）。
      空天地一体
      NTN Integration
      卫星（NTN）与地面网络在同一核心架构下统一接入与管理，实现真正的全球无缝覆盖。
      数字孪生网络
      Digital Twin Network
      为物理网络建立实时数字镜像，先在孪生体中仿真验证再下发，实现预测式运维与零接触运营。
      内生安全
      Intrinsic Security
      安全从"打补丁"变为架构内生：零信任、隐私计算、抗量子加密成为核心网默认能力。

      从 5G 到 6G 的架构延续性

      6G 不会推倒重来，而是沿着 5G 的方向继续深化：

      • 服务化更彻底：从 NF 服务化走向"网络即服务（NaaS）"，能力进一步原子化、API 化。
      • 云原生更下沉：算力与网络融合，UPF/边缘进一步分布式、随业务流动。
      • 智能更内生：数据采集、模型训练、推理闭环成为架构标准组件。
      • 覆盖更立体：从地面平面网络走向"空-天-地-海"三维网络。
      演进延续：解耦 → 服务化 → 智能化
      4G 全IP+CUPS
      5G SBA+云原生
      6G AI原生+一体化
      展望小结如果说 5G 让核心网"上了云"，6G 的目标是让核心网"长出大脑"——网络能感知、会思考、自运营，并把通信、计算、感知、覆盖融为一体。
